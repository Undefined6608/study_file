008    import java.util.concurrent.*;
009    import java.util.concurrent.atomic.*;

012    /**
013     * An implementation of {@link ReadWriteLock} supporting similar
014     * semantics to {@link ReentrantLock}.
015     * <p>This class has the following properties:
016     *
017     * <ul>
018     * <li><b>Acquisition order</b>
019     *
020     * <p> This class does not impose a reader or writer preference
021     * ordering for lock access.  However, it does support an optional
022     * <em>fairness</em> policy.  When constructed as fair, threads
023     * contend for entry using an approximately arrival-order policy. When
024     * the write lock is released either the longest-waiting single writer
025     * will be assigned the write lock, or if there is a reader waiting
026     * longer than any writer, the set of readers will be assigned the
027     * read lock.  When constructed as non-fair, the order of entry to the
028     * lock need not be in arrival order.  In either case, if readers are
029     * active and a writer enters the lock then no subsequent readers will
030     * be granted the read lock until after that writer has acquired and
031     * released the write lock.
032     * 
033     * <li><b>Reentrancy</b>
034     * <p>This lock allows both readers and writers to reacquire read or
035     * write locks in the style of a {@link ReentrantLock}. Readers are not
036     * allowed until all write locks held by the writing thread have been
037     * released.  
038     * <p>Additionally, a writer can acquire the read lock - but not vice-versa.
039     * Among other applications, reentrancy can be useful when
040     * write locks are held during calls or callbacks to methods that
041     * perform reads under read locks. 
042     * If a reader tries to acquire the write lock it will never succeed.
043     * 
044     * <li><b>Lock downgrading</b>
045     * <p>Reentrancy also allows downgrading from the write lock to a read lock,
046     * by acquiring the write lock, then the read lock and then releasing the
047     * write lock. However, upgrading from a read lock to the write lock, is
048     * <b>not</b> possible.
049     *
050     * <li><b>Interruption of lock acquisition</b>
051     * <p>The read lock and write lock both support interruption during lock
052     * acquisition.
053     *
054     * <li><b>{@link Condition} support</b>
055     * <p>The write lock provides a {@link Condition} implementation that
056     * behaves in the same way, with respect to the write lock, as the 
057     * {@link Condition} implementation provided by
058     * {@link ReentrantLock#newCondition} does for {@link ReentrantLock}.
059     * This {@link Condition} can, of course, only be used with the write lock.
060     * <p>The read lock does not support a {@link Condition} and
061     * <tt>readLock().newCondition()</tt> throws 
062     * <tt>UnsupportedOperationException</tt>.
063     *
064     * <li><b>Instrumentation</b>
065     * <P> This class supports methods to determine whether locks
066     * are held or contended. These methods are designed for monitoring
067     * system state, not for synchronization control.
068     * </ul>
069     *
070     * <p> Serialization of this class behaves in the same way as built-in
071     * locks: a deserialized lock is in the unlocked state, regardless of
072     * its state when serialized.
073     *
074     * <p><b>Sample usages</b>. Here is a code sketch showing how to exploit
075     * reentrancy to perform lock downgrading after updating a cache (exception
076     * handling is elided for simplicity):
077     * <pre>
078     * class CachedData {
079     *   Object data;
080     *   volatile boolean cacheValid;
081     *   ReentrantReadWriteLock rwl = new ReentrantReadWriteLock();
082     *
083     *   void processCachedData() {
084     *     rwl.readLock().lock();
085     *     if (!cacheValid) {
086     *        // upgrade lock manually
087     *        rwl.readLock().unlock();   // must unlock first to obtain writelock
088     *        rwl.writeLock().lock();
089     *        if (!cacheValid) { // recheck
090     *          data = ...
091     *          cacheValid = true;
092     *        }
093     *        // downgrade lock
094     *        rwl.readLock().lock();  // reacquire read without giving up write lock
095     *        rwl.writeLock().unlock(); // unlock write, still hold read
096     *     }
097     *
098     *     use(data);
099     *     rwl.readLock().unlock();
100     *   }
101     * }
102     * </pre>
103     *
104     * ReentrantReadWriteLocks can be used to improve concurrency in some
105     * uses of some kinds of Collections. This is typically worthwhile
106     * only when the collections are expected to be large, accessed by
107     * more reader threads than writer threads, and entail operations with
108     * overhead that outweighs synchronization overhead. For example, here
109     * is a class using a TreeMap that is expected to be large and 
110     * concurrently accessed.
111     *
112     * <pre>
113     * class RWDictionary {
114     *    private final Map&lt;String, Data&gt;  m = new TreeMap&lt;String, Data&gt;();
115     *    private final ReentrantReadWriteLock rwl = new ReentrantReadWriteLock();
116     *    private final Lock r = rwl.readLock();
117     *    private final Lock w = rwl.writeLock();
118     *
119     *    public Data get(String key) {
120     *        r.lock(); try { return m.get(key); } finally { r.unlock(); }
121     *    }
122     *    public String[] allKeys() {
123     *        r.lock(); try { return m.keySet().toArray(); } finally { r.unlock(); }
124     *    }
125     *    public Data put(String key, Data value) {
126     *        w.lock(); try { return m.put(key, value); } finally { w.unlock(); }
127     *    }
128     *    public void clear() {
129     *        w.lock(); try { m.clear(); } finally { w.unlock(); }
130     *    }
131     * }
132     * </pre>
133     * 
134     *
135     * <h3>Implementation Notes</h3>
136     *
137     * <p>A reentrant write lock intrinsically defines an owner and can
138     * only be released by the thread that acquired it.  In contrast, in
139     * this implementation, the read lock has no concept of ownership, and
140     * there is no requirement that the thread releasing a read lock is
141     * the same as the one that acquired it.  However, this property is
142     * not guaranteed to hold in future implementations of this class.
143     *
144     * <p> This lock supports a maximum of 65536 recursive write locks
145     * and 65536 read locks. Attempts to exceed these limits result in
146     * {@link Error} throws from locking methods.
147     *
148     * @since 1.5
149     * @author Doug Lea
150     *
151     */
152    public class ReentrantReadWriteLock implements ReadWriteLock, java.io.Serializable  {
153        private static final long serialVersionUID = -6992448646407690164L;
154        /** Inner class providing readlock */
155        private final ReentrantReadWriteLock.ReadLock readerLock;
156        /** Inner class providing writelock */
157        private final ReentrantReadWriteLock.WriteLock writerLock;
158        /** Performs all synchronization mechanics */
159        private final Sync sync;

161        /**
162         * Creates a new <tt>ReentrantReadWriteLock</tt> with
163         * default ordering properties.
164         */
165        public ReentrantReadWriteLock() {
166            sync = new NonfairSync();
167            readerLock = new ReadLock(this);
168            writerLock = new WriteLock(this);
169        }
170    
171        /**
172         * Creates a new <tt>ReentrantReadWriteLock</tt> with
173         * the given fairness policy.
174         *
175         * @param fair true if this lock should use a fair ordering policy
176         */
177        public ReentrantReadWriteLock(boolean fair) {
178            sync = (fair)? new FairSync() : new NonfairSync();
179            readerLock = new ReadLock(this);
180            writerLock = new WriteLock(this);
181        }

183        public ReentrantReadWriteLock.WriteLock writeLock() { return writerLock; }
184        public ReentrantReadWriteLock.ReadLock  readLock()  { return readerLock; }

186        /* 
187         * Read vs write count extraction constants and functions.
188         * Lock state is logically divided into two shorts: The lower
189         * one representing the exclusive (writer) lock hold count,
190         * and the upper the shared (reader) hold count.
191         */
192        
193        static final int SHARED_SHIFT   = 16;
194        static final int SHARED_UNIT    = (1 << SHARED_SHIFT);
195        static final int EXCLUSIVE_MASK = (1 << SHARED_SHIFT) - 1;
196        
197        /** Returns the number of shared holds represented in count  */
198        static int sharedCount(int c)    { return c >>> SHARED_SHIFT; }
199        /** Returns the number of exclusive holds represented in count  */
200        static int exclusiveCount(int c) { return c & EXCLUSIVE_MASK; }
201    
202        /** 
203         * Synchronization implementation for ReentrantReadWriteLock.
204         * Subclassed into fair and nonfair versions.
205         */
206        abstract static class Sync extends AbstractQueuedSynchronizer {
207            /** Current (exclusive) owner thread */
208            transient Thread owner;
209    
210            /**
211             * Perform write lock. Allows fast path in non-fair version.
212             */
213            abstract void wlock();
214    
215            /** 
216             * Perform non-fair tryLock for write.  tryAcquire is
217             * implemented in subclasses, but both versions need nonfair
218             * try for trylock method
219             */
220            final boolean nonfairTryAcquire(int acquires) {
221                // mask out readlocks if called from condition methods
222                acquires = exclusiveCount(acquires);
223                Thread current = Thread.currentThread();
224                int c = getState();
225                int w = exclusiveCount(c);
226                if (w + acquires >= SHARED_UNIT)
227                    throw new Error("Maximum lock count exceeded");
228                if (c != 0 && (w == 0 || current != owner))
229                    return false;
230                if (!compareAndSetState(c, c + acquires)) 
231                    return false;
232                owner = current;
233                return true;
234            }
235    
236            /** 
237             * Perform nonfair tryLock for read. 
238             */
239            final int nonfairTryAcquireShared(int acquires) {
240                for (;;) {
241                    int c = getState();
242                    int nextc = c + (acquires << SHARED_SHIFT);
243                    if (nextc < c)
244                        throw new Error("Maximum lock count exceeded");
245                    if (exclusiveCount(c) != 0 && 
246                        owner != Thread.currentThread())
247                        return -1;
248                    if (compareAndSetState(c, nextc)) 
249                        return 1;
250                    // Recheck count if lost CAS
251                }
252            }
253    
254            protected final boolean tryRelease(int releases) {
255                Thread current = Thread.currentThread();
256                int c = getState();
257                if (owner != current)
258                    throw new IllegalMonitorStateException();
259                int nextc = c - releases;
260                boolean free = false;
261                if (exclusiveCount(c) == releases) {
262                    free = true;
263                    owner = null;
264                }
265                setState(nextc);
266                return free;
267            }
268    
269            protected final boolean tryReleaseShared(int releases) {
270                for (;;) {
271                    int c = getState();
272                    int nextc = c - (releases << SHARED_SHIFT);
273                    if (nextc < 0)
274                        throw new IllegalMonitorStateException();
275                    if (compareAndSetState(c, nextc)) 
276                        return nextc == 0;
277                }
278            }
279        
280            protected final boolean isHeldExclusively() {
281                return exclusiveCount(getState()) != 0 && 
282                    owner == Thread.currentThread();
283            }
284    
285            // Methods relayed to outer class
286            
287            final ConditionObject newCondition() { 
288                return new ConditionObject(); 
289            }
290    
291            final Thread getOwner() {
292                int c = exclusiveCount(getState());
293                Thread o = owner;
294                return (c == 0)? null : o;
295            }
296            
297            final int getReadLockCount() {
298                return sharedCount(getState());
299            }
300            
301            final boolean isWriteLocked() {
302                return exclusiveCount(getState()) != 0;
303            }
304    
305            final int getWriteHoldCount() {
306                int c = exclusiveCount(getState());
307                Thread o = owner;
308                return (o == Thread.currentThread())? c : 0;
309            }
310    
311            /**
312             * Reconstitute this lock instance from a stream
313             * @param s the stream
314             */
315            private void readObject(java.io.ObjectInputStream s)
316                throws java.io.IOException, ClassNotFoundException {
317                s.defaultReadObject();
318                setState(0); // reset to unlocked state
319            }
320    
321            final int getCount() { return getState(); }
322        }
323    
324        /** 
325         * Nonfair version of Sync
326         */
327        final static class NonfairSync extends Sync {
328            protected final boolean tryAcquire(int acquires) { 
329                return nonfairTryAcquire(acquires);
330            }
331    
332            protected final int tryAcquireShared(int acquires) {
333                return nonfairTryAcquireShared(acquires);
334            }
335    
336            // Use fastpath for main write lock method
337            final void wlock() {
338                if (compareAndSetState(0, 1))
339                    owner = Thread.currentThread();
340                else
341                    acquire(1);
342            }
343        }
344    
345        /** 
346         * Fair version of Sync
347         */
348        final static class FairSync extends Sync {
349            protected final boolean tryAcquire(int acquires) { 
350                // mask out readlocks if called from condition methods
351                acquires = exclusiveCount(acquires);
352                Thread current = Thread.currentThread();
353                Thread first;
354                int c = getState();
355                int w = exclusiveCount(c);
356                if (w + acquires >= SHARED_UNIT)
357                    throw new Error("Maximum lock count exceeded");
358                if ((w == 0 || current != owner) &&
359                    (c != 0 || 
360                     ((first = getFirstQueuedThread()) != null && 
361                      first != current)))
362                    return false;
363                if (!compareAndSetState(c, c + acquires)) 
364                    return false;
365                owner = current;
366                return true;
367            }
368    
369            protected final int tryAcquireShared(int acquires) {
370                Thread current = Thread.currentThread();
371                for (;;) {
372                    Thread first = getFirstQueuedThread();
373                    if (first != null && first != current)
374                        return -1;
375                    int c = getState();
376                    int nextc = c + (acquires << SHARED_SHIFT);
377                    if (nextc < c)
378                        throw new Error("Maximum lock count exceeded");
379                    if (exclusiveCount(c) != 0 && 
380                        owner != Thread.currentThread())
381                        return -1;
382                    if (compareAndSetState(c, nextc)) 
383                        return 1;
384                    // Recheck count if lost CAS
385                }
386            }
387    
388            final void wlock() { // no fast path
389                acquire(1);
390            }
391        }

393        /**
394         * The lock returned by method {@link ReentrantReadWriteLock#readLock}.
395         */
396        public static class ReadLock implements Lock, java.io.Serializable  {
397            private static final long serialVersionUID = -5992448646407690164L;
398            private final Sync sync;
399            
400            /** 
401             * Constructor for use by subclasses.
402             * @param lock the outer lock object
403             * @throws NullPointerException if lock null
404             */
405            protected ReadLock(ReentrantReadWriteLock lock) {
406                sync = lock.sync;
407            }
408    
409            /**
410             * Acquires the shared lock. 
411             *
412             * <p>Acquires the lock if it is not held exclusively by
413             * another thread and returns immediately.
414             *
415             * <p>If the lock is held exclusively by another thread then
416             * the current thread becomes disabled for thread scheduling
417             * purposes and lies dormant until the lock has been acquired.
418             */
419            public void lock() { 
420                sync.acquireShared(1);
421            }
422    
423            /**
424             * Acquires the shared lock unless the current thread is 
425             * {@link Thread#interrupt interrupted}.
426             *
427             * <p>Acquires the shared lock if it is not held exclusively
428             * by another thread and returns immediately.
429             *
430             * <p>If the lock is held by another thread then the
431             * current thread becomes disabled for thread scheduling 
432             * purposes and lies dormant until one of two things happens:
433             *
434             * <ul>
435             *
436             * <li>The lock is acquired by the current thread; or
437             *
438             * <li>Some other thread {@link Thread#interrupt interrupts}
439             * the current thread.
440             *
441             * </ul>
442             *
443             * <p>If the current thread:
444             *
445             * <ul>
446             *
447             * <li>has its interrupted status set on entry to this method; or 
448             *
449             * <li>is {@link Thread#interrupt interrupted} while acquiring 
450             * the lock,
451             *
452             * </ul>
453             *
454             * then {@link InterruptedException} is thrown and the current
455             * thread's interrupted status is cleared.
456             *
457             * <p>In this implementation, as this method is an explicit
458             * interruption point, preference is given to responding to
459             * the interrupt over normal or reentrant acquisition of the
460             * lock.
461             *
462             * @throws InterruptedException if the current thread is interrupted
463             */
464            public void lockInterruptibly() throws InterruptedException {
465                sync.acquireSharedInterruptibly(1);
466            }
467    
468            /**
469             * Acquires the shared lock only if it is not held exclusively by
470             * another thread at the time of invocation.
471             *
472             * <p>Acquires the lock if it is not held exclusively by
473             * another thread and returns immediately with the value
474             * <tt>true</tt>. Even when this lock has been set to use a
475             * fair ordering policy, a call to <tt>tryLock()</tt>
476             * <em>will</em> immediately acquire the lock if it is
477             * available, whether or not other threads are currently
478             * waiting for the lock.  This &quot;barging&quot; behavior
479             * can be useful in certain circumstances, even though it
480             * breaks fairness. If you want to honor the fairness setting
481             * for this lock, then use {@link #tryLock(long, TimeUnit)
482             * tryLock(0, TimeUnit.SECONDS) } which is almost equivalent
483             * (it also detects interruption).
484             *
485             * <p>If the lock is held exclusively by another thread then
486             * this method will return immediately with the value
487             * <tt>false</tt>.
488             *
489             * @return <tt>true</tt> if the lock was acquired.
490             */
491            public  boolean tryLock() {
492                return sync.nonfairTryAcquireShared(1) >= 0;
493            }
494    
495            /**
496             * Acquires the shared lock if it is not held exclusively by
497             * another thread within the given waiting time and the
498             * current thread has not been {@link Thread#interrupt
499             * interrupted}.
500             *
501             * <p>Acquires the lock if it is not held exclusively by
502             * another thread and returns immediately with the value
503             * <tt>true</tt>. If this lock has been set to use a fair
504             * ordering policy then an available lock <em>will not</em> be
505             * acquired if any other threads are waiting for the
506             * lock. This is in contrast to the {@link #tryLock()}
507             * method. If you want a timed <tt>tryLock</tt> that does
508             * permit barging on a fair lock then combine the timed and
509             * un-timed forms together:
510             *
511             * <pre>if (lock.tryLock() || lock.tryLock(timeout, unit) ) { ... }
512             * </pre>
513             *
514             * <p>If the lock is held exclusively by another thread then the
515             * current thread becomes disabled for thread scheduling 
516             * purposes and lies dormant until one of three things happens:
517             *
518             * <ul>
519             *
520             * <li>The lock is acquired by the current thread; or
521             *
522             * <li>Some other thread {@link Thread#interrupt interrupts} the current
523             * thread; or
524             *
525             * <li>The specified waiting time elapses
526             *
527             * </ul>
528             *
529             * <p>If the lock is acquired then the value <tt>true</tt> is
530             * returned.
531             *
532             * <p>If the current thread:
533             *
534             * <ul>
535             *
536             * <li>has its interrupted status set on entry to this method; or 
537             *
538             * <li>is {@link Thread#interrupt interrupted} while acquiring
539             * the lock,
540             *
541             * </ul> then {@link InterruptedException} is thrown and the
542             * current thread's interrupted status is cleared.
543             *
544             * <p>If the specified waiting time elapses then the value
545             * <tt>false</tt> is returned.  If the time is less than or
546             * equal to zero, the method will not wait at all.
547             *
548             * <p>In this implementation, as this method is an explicit
549             * interruption point, preference is given to responding to
550             * the interrupt over normal or reentrant acquisition of the
551             * lock, and over reporting the elapse of the waiting time.
552             *
553             * @param timeout the time to wait for the lock
554             * @param unit the time unit of the timeout argument
555             *
556             * @return <tt>true</tt> if the lock was acquired.
557             *
558             * @throws InterruptedException if the current thread is interrupted
559             * @throws NullPointerException if unit is null
560             *
561             */
562            public boolean tryLock(long timeout, TimeUnit unit) throws InterruptedException {
563                return sync.tryAcquireSharedNanos(1, unit.toNanos(timeout));
564            }
565    
566            /**
567             * Attempts to release this lock.  
568             *
569             * <p> If the number of readers is now zero then the lock
570             * is made available for other lock attempts.
571             */
572            public  void unlock() {
573                sync.releaseShared(1);
574            }
575    
576            /**
577             * Throws UnsupportedOperationException because ReadLocks
578             * do not support conditions.
579             * @return A new {@link Condition} instance for this <tt>Lock</tt> 
580             * instance.
581             * @throws UnsupportedOperationException always
582             */
583            public Condition newCondition() {
584                throw new UnsupportedOperationException();
585            }

587            /**
588             * Returns a string identifying this lock, as well as its lock state.
589             * The state, in brackets, includes the String 
590             * &quot;Read locks =&quot; followed by the number of held
591             * read locks.
592             * @return a string identifying this lock, as well as its lock state.
593             */
594            public String toString() {
595                int r = sync.getReadLockCount();
596                return super.toString() + 
597                    "[Read locks = " + r + "]";
598            }

600        }

609            /**

618            /**
619             * Acquire the lock. 
620             *
621             * <p>Acquires the lock if it is not held by another thread
622             * and returns immediately, setting the lock hold count to
623             * one.
624             *

638            /**

641             *
642             * <p>Acquires the lock if it is not held by another thread
643             * and returns immediately, setting the lock hold count to
644             * one.
645             *
646             * <p>If the current thread already holds this lock then the
647             * hold count is incremented by one and the method returns
648             * immediately.
649             *
650             * <p>If the lock is held by another thread then the current
651             * thread becomes disabled for thread scheduling purposes and
652             * lies dormant until one of two things happens:
653             *

655             *
656             * <li>The lock is acquired by the current thread; or
657             *
658             * <li>Some other thread {@link Thread#interrupt interrupts}
659             * the current thread.
660             *
661             * </ul>
662             *
663             * <p>If the lock is acquired by the current thread then the
664             * lock hold count is set to one.
665             *
666             * <p>If the current thread:
667             *
668             * <ul>
669             *
670             * <li>has its interrupted status set on entry to this method;
671             * or
672             *
673             * <li>is {@link Thread#interrupt interrupted} while acquiring
674             * the lock,
675             *
676             * </ul>
677             *
678             * then {@link InterruptedException} is thrown and the current
679             * thread's interrupted status is cleared.
680             *

692            /**
693             * Acquires the lock only if it is not held by another thread
694             * at the time of invocation.
695             *
696             * <p>Acquires the lock if it is not held by another thread
697             * and returns immediately with the value <tt>true</tt>,
698             * setting the lock hold count to one. Even when this lock has
699             * been set to use a fair ordering policy, a call to
700             * <tt>tryLock()</tt> <em>will</em> immediately acquire the
701             * lock if it is available, whether or not other threads are
702             * currently waiting for the lock.  This &quot;barging&quot;
703             * behavior can be useful in certain circumstances, even
704             * though it breaks fairness. If you want to honor the
705             * fairness setting for this lock, then use {@link
706             * #tryLock(long, TimeUnit) tryLock(0, TimeUnit.SECONDS) }
707             * which is almost equivalent (it also detects interruption).
708             *
709             * <p> If the current thread already holds this lock then the
710             * hold count is incremented by one and the method returns
711             * <tt>true</tt>.
712             *
713             * <p>If the lock is held by another thread then this method
714             * will return immediately with the value <tt>false</tt>.
715             *
716             * @return <tt>true</tt> if the lock was free and was acquired by the
717             * current thread, or the lock was already held by the current thread; and
718             * <tt>false</tt> otherwise.
719             */
720            public boolean tryLock( ) {
721                return sync.nonfairTryAcquire(1);
722            }

724            /**
725             * Acquires the lock if it is not held by another thread
726             * within the given waiting time and the current thread has
727             * not been {@link Thread#interrupt interrupted}.
728             *
729             * <p>Acquires the lock if it is not held by another thread
730             * and returns immediately with the value <tt>true</tt>,
731             * setting the lock hold count to one. If this lock has been
732             * set to use a fair ordering policy then an available lock
733             * <em>will not</em> be acquired if any other threads are
734             * waiting for the lock. This is in contrast to the {@link
735             * #tryLock()} method. If you want a timed <tt>tryLock</tt>
736             * that does permit barging on a fair lock then combine the
737             * timed and un-timed forms together:
738             *
739             * <pre>if (lock.tryLock() || lock.tryLock(timeout, unit) ) { ... }
740             * </pre>
741             *
742             * <p>If the current thread already holds this lock then the
743             * hold count is incremented by one and the method returns
744             * <tt>true</tt>.
745             *
746             * <p>If the lock is held by another thread then the current
747             * thread becomes disabled for thread scheduling purposes and
748             * lies dormant until one of three things happens:
749             *
750             * <ul>
751             *
752             * <li>The lock is acquired by the current thread; or
753             *
754             * <li>Some other thread {@link Thread#interrupt interrupts}
755             * the current thread; or
756             *
757             * <li>The specified waiting time elapses
758             *
759             * </ul>
760             *
761             * <p>If the lock is acquired then the value <tt>true</tt> is
762             * returned and the lock hold count is set to one.
763             *
764             * <p>If the current thread:
765             *
766             * <ul>
767             *
768             * <li>has its interrupted status set on entry to this method;
769             * or
770             *
771             * <li>is {@link Thread#interrupt interrupted} while acquiring
772             * the lock,
773             *
774             * </ul> 
775             *
776             * then {@link InterruptedException} is thrown and the current
777             * thread's interrupted status is cleared.
778             *
779             * <p>If the specified waiting time elapses then the value
780             * <tt>false</tt> is returned.  If the time is less than or
781             * equal to zero, the method will not wait at all.
782             *
783             * <p>In this implementation, as this method is an explicit
784             * interruption point, preference is given to responding to
785             * the interrupt over normal or reentrant acquisition of the
786             * lock, and over reporting the elapse of the waiting time.
787             *
788             * @param timeout the time to wait for the lock
789             * @param unit the time unit of the timeout argument
790             *
791             * @return <tt>true</tt> if the lock was free and was acquired
792             * by the current thread, or the lock was already held by the
793             * current thread; and <tt>false</tt> if the waiting time
794             * elapsed before the lock could be acquired.
795             *
796             * @throws InterruptedException if the current thread is interrupted
797             * @throws NullPointerException if unit is null
798             *
799             */
800            public boolean tryLock(long timeout, TimeUnit unit) throws InterruptedException {
801                return sync.tryAcquireNanos(1, unit.toNanos(timeout));
802            }

807             * <p>If the current thread is the holder of this lock then
808             * the hold count is decremented. If the hold count is now

810             * not the holder of this lock then {@link
811             * IllegalMonitorStateException} is thrown.

820             * Returns a {@link Condition} instance for use with this
821             * {@link Lock} instance. 
822             * <p>The returned {@link Condition} instance supports the same
823             * usages as do the {@link Object} monitor methods ({@link
824             * Object#wait() wait}, {@link Object#notify notify}, and {@link
825             * Object#notifyAll notifyAll}) when used with the built-in
826             * monitor lock.
827             *
828             * <ul>
829             *
830             * <li>If this write lock is not held when any {@link
831             * Condition} method is called then an {@link
832             * IllegalMonitorStateException} is thrown.  (Read locks are
833             * held independently of write locks, so are not checked or
834             * affected. However it is essentially always an error to
835             * invoke a condition waiting method when the current thread
836             * has also acquired read locks, since other threads that
837             * could unblock it will not be able to access the write
838             * lock.)
839             *
840             * <li>When the condition {@link Condition#await() waiting}
841             * methods are called the write lock is released and, before
842             * they return, the write lock is reacquired and the lock hold
843             * count restored to what it was when the method was called.
844             *
845             * <li>If a thread is {@link Thread#interrupt interrupted} while
846             * waiting then the wait will terminate, an {@link
847             * InterruptedException} will be thrown, and the thread's
848             * interrupted status will be cleared.
849             *
850             * <li> Waiting threads are signalled in FIFO order
851             *
852             * <li>The ordering of lock reacquisition for threads returning
853             * from waiting methods is the same as for threads initially
854             * acquiring the lock, which is in the default case not specified,
855             * but for <em>fair</em> locks favors those threads that have been
856             * waiting the longest.
857             * 
858             * </ul>
859             * @return the Condition object
860             */
861            public Condition newCondition() { 
862                return sync.newCondition();
863            }
