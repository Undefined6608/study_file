013    /**
014     * Provides a framework for implementing blocking locks and related
015     * synchronizers (semaphores, events, etc) that rely on
016     * first-in-first-out (FIFO) wait queues.  This class is designed to
017     * be a useful basis for most kinds of synchronizers that rely on a
018     * single atomic <tt>int</tt> value to represent state. Subclasses
019     * must define the protected methods that change this state, and which
020     * define what that state means in terms of this object being acquired
021     * or released.  Given these, the other methods in this class carry
022     * out all queuing and blocking mechanics. Subclasses can maintain
023     * other state fields, but only the atomically updated <tt>int</tt>
024     * value manipulated using methods {@link #getState}, {@link
025     * #setState} and {@link #compareAndSetState} is tracked with respect
026     * to synchronization.
027     *
028     * <p>Subclasses should be defined as non-public internal helper
029     * classes that are used to implement the synchronization properties
030     * of their enclosing class.  Class
031     * <tt>AbstractQueuedSynchronizer</tt> does not implement any
032     * synchronization interface.  Instead it defines methods such as
033     * {@link #acquireInterruptibly} that can be invoked as
034     * appropriate by concrete locks and related synchronizers to
035     * implement their public methods. 
036     *
037     * <p>This class supports either or both a default <em>exclusive</em>
038     * mode and a <em>shared</em> mode. When acquired in exclusive mode,
039     * attempted acquires by other threads cannot succeed. Shared mode
040     * acquires by multiple threads may (but need not) succeed. This class
041     * does not &quot;understand&quot; these differences except in the
042     * mechanical sense that when a shared mode acquire succeeds, the next
043     * waiting thread (if one exists) must also determine whether it can
044     * acquire as well. Threads waiting in the different modes share the
045     * same FIFO queue. Usually, implementation subclasses support only
046     * one of these modes, but both can come into play for example in a
047     * {@link ReadWriteLock}. Subclasses that support only exclusive or
048     * only shared modes need not define the methods supporting the unused mode.
049     *
050     * <p>This class defines a nested {@link ConditionObject} class that
051     * can be used as a {@link Condition} implementation by subclasses
052     * supporting exclusive mode for which method {@link
053     * #isHeldExclusively} reports whether synchronization is exclusively
054     * held with respect to the current thread, method {@link #release}
055     * invoked with the current {@link #getState} value fully releases
056     * this object, and {@link #acquire}, given this saved state value,
057     * eventually restores this object to its previous acquired state.  No
058     * <tt>AbstractQueuedSynchronizer</tt> method otherwise creates such a
059     * condition, so if this constraint cannot be met, do not use it.  The
060     * behavior of {@link ConditionObject} depends of course on the
061     * semantics of its synchronizer implementation.
062     * 
063     * <p> This class provides inspection, instrumentation, and monitoring
064     * methods for the internal queue, as well as similar methods for
065     * condition objects. These can be exported as desired into classes
066     * using an <tt>AbstractQueuedSynchronizer</tt> for their
067     * synchronization mechanics.
068     *
069     * <p> Serialization of this class stores only the underlying atomic
070     * integer maintaining state, so deserialized objects have empty
071     * thread queues. Typical subclasses requiring serializability will
072     * define a <tt>readObject</tt> method that restores this to a known
073     * initial state upon deserialization.
074     *
075     * <h3>Usage</h3>
076     *
077     * <p> To use this class as the basis of a synchronizer, redefine the
078     * following methods, as applicable, by inspecting and/or modifying
079     * the synchronization state using {@link #getState}, {@link
080     * #setState} and/or {@link #compareAndSetState}: 
081     *
082     * <ul>
083     * <li> {@link #tryAcquire}
084     * <li> {@link #tryRelease}
085     * <li> {@link #tryAcquireShared}
086     * <li> {@link #tryReleaseShared}
087     * <li> {@link #isHeldExclusively}
088     *</ul>
089     *
090     * Each of these methods by default throws {@link
091     * UnsupportedOperationException}.  Implementations of these methods
092     * must be internally thread-safe, and should in general be short and
093     * not block. Defining these methods is the <em>only</em> supported
094     * means of using this class. All other methods are declared
095     * <tt>final</tt> because they cannot be independently varied.
096     *
097     * <p> Even though this class is based on an internal FIFO queue, it
098     * does not automatically enforce FIFO acquisition policies.  The core
099     * of exclusive synchronization takes the form:
100     *
101     * <pre>
102     * Acquire:
103     *     while (!tryAcquire(arg)) {
104     *        <em>enqueue thread if it is not already queued</em>;
105     *        <em>possibly block current thread</em>;
106     *     }
107     *
108     * Release:
109     *     if (tryRelease(arg))
110     *        <em>unblock the first queued thread</em>;
111     * </pre>
112     *
113     * (Shared mode is similar but may involve cascading signals.)
114     *
115     * <p> Because checks in acquire are invoked before enqueuing, a newly
116     * acquiring thread may <em>barge</em> ahead of others that are
117     * blocked and queued. However, you can, if desired, define
118     * <tt>tryAcquire</tt> and/or <tt>tryAcquireShared</tt> to disable
119     * barging by internally invoking one or more of the inspection
120     * methods. In particular, a strict FIFO lock can define
121     * <tt>tryAcquire</tt> to immediately return <tt>false</tt> if {@link
122     * #getFirstQueuedThread} does not return the current thread.  A
123     * normally preferable non-strict fair version can immediately return
124     * <tt>false</tt> only if {@link #hasQueuedThreads} returns
125     * <tt>true</tt> and <tt>getFirstQueuedThread</tt> is not the current
126     * thread; or equivalently, that <tt>getFirstQueuedThread</tt> is both
127     * non-null and not the current thread.  Further variations are
128     * possible.
129     *
130     * <p> Throughput and scalability are generally highest for the
131     * default barging (also known as <em>greedy</em>,
132     * <em>renouncement</em>, and <em>convoy-avoidance</em>) strategy.
133     * While this is not guaranteed to be fair or starvation-free, earlier
134     * queued threads are allowed to recontend before later queued
135     * threads, and each recontention has an unbiased chance to succeed
136     * against incoming threads.  Also, while acquires do not
137     * &quot;spin&quot; in the usual sense, they may perform multiple
138     * invocations of <tt>tryAcquire</tt> interspersed with other
139     * computations before blocking.  This gives most of the benefits of
140     * spins when exclusive synchronization is only briefly held, without
141     * most of the liabilities when it isn't. If so desired, you can
142     * augment this by preceding calls to acquire methods with
143     * "fast-path" checks, possibly prechecking {@link #hasContended}
144     * and/or {@link #hasQueuedThreads} to only do so if the synchronizer
145     * is likely not to be contended.
146     *
147     * <p> This class provides an efficient and scalable basis for
148     * synchronization in part by specializing its range of use to
149     * synchronizers that can rely on <tt>int</tt> state, acquire, and
150     * release parameters, and an internal FIFO wait queue. When this does
151     * not suffice, you can build synchronizers from a lower level using
152     * {@link java.util.concurrent.atomic atomic} classes, your own custom
153     * {@link java.util.Queue} classes, and {@link LockSupport} blocking
154     * support.
155     * 
156     * <h3>Usage Examples</h3>
157     *
158     * <p>Here is a non-reentrant mutual exclusion lock class that uses
159     * the value zero to represent the unlocked state, and one to
160     * represent the locked state. It also supports conditions and exposes
161     * one of the instrumentation methods:
162     *
163     * <pre>
164     * class Mutex implements Lock, java.io.Serializable {
165     *
166     *    // Our internal helper class
167     *    private static class Sync extends AbstractQueuedSynchronizer {
168     *      // Report whether in locked state
169     *      protected boolean isHeldExclusively() { 
170     *        return getState() == 1; 
171     *      }
172     *
173     *      // Acquire the lock if state is zero
174     *      public boolean tryAcquire(int acquires) {
175     *        assert acquires == 1; // Otherwise unused
176     *        return compareAndSetState(0, 1);
177     *      }
178     *
179     *      // Release the lock by setting state to zero
180     *      protected boolean tryRelease(int releases) {
181     *        assert releases == 1; // Otherwise unused
182     *        if (getState() == 0) throw new IllegalMonitorStateException();
183     *        setState(0);
184     *        return true;
185     *      }
186     *       
187     *      // Provide a Condition
188     *      Condition newCondition() { return new ConditionObject(); }
189     *
190     *      // Deserialize properly
191     *      private void readObject(ObjectInputStream s) throws IOException, ClassNotFoundException {
192     *        s.defaultReadObject();
193     *        setState(0); // reset to unlocked state
194     *      }
195     *    }
196     *
197     *    // The sync object does all the hard work. We just forward to it.
198     *    private final Sync sync = new Sync();
199     *
200     *    public void lock()                { sync.acquire(1); }
201     *    public boolean tryLock()          { return sync.tryAcquire(1); }
202     *    public void unlock()              { sync.release(1); }
203     *    public Condition newCondition()   { return sync.newCondition(); }
204     *    public boolean isLocked()         { return sync.isHeldExclusively(); }
205     *    public boolean hasQueuedThreads() { return sync.hasQueuedThreads(); }
206     *    public void lockInterruptibly() throws InterruptedException { 
207     *      sync.acquireInterruptibly(1);
208     *    }
209     *    public boolean tryLock(long timeout, TimeUnit unit) throws InterruptedException {
210     *      return sync.tryAcquireNanos(1, unit.toNanos(timeout));
211     *    }
212     * }
213     * </pre>
214     *
215     * <p> Here is a latch class that is like a {@link CountDownLatch}
216     * except that it only requires a single <tt>signal</tt> to
217     * fire. Because a latch is non-exclusive, it uses the <tt>shared</tt>
218     * acquire and release methods.
219     *
220     * <pre>
221     * class BooleanLatch {
222     *
223     *    private static class Sync extends AbstractQueuedSynchronizer {
224     *      boolean isSignalled() { return getState() != 0; }
225     *
226     *      protected int tryAcquireShared(int ignore) {
227     *        return isSignalled()? 1 : -1;
228     *      }
229     *        
230     *      protected boolean tryReleaseShared(int ignore) {
231     *        setState(1);
232     *        return true;
233     *      }
234     *    }
235     *
236     *    private final Sync sync = new Sync();
237     *    public boolean isSignalled() { return sync.isSignalled(); }
238     *    public void signal()         { sync.releaseShared(1); }
239     *    public void await() throws InterruptedException {
240     *      sync.acquireSharedInterruptibly(1);
241     *    }
242     * }
243     *
244     * </pre>

258        /**
259         * Wait queue node class.
260         *
261         * <p> The wait queue is a variant of a "CLH" (Craig, Landin, and
262         * Hagersten) lock queue. CLH locks are normally used for
263         * spinlocks.  We instead use them for blocking synchronizers, but
264         * use the same basic tactic of holding some of the control
265         * information about a thread in the predecessor of its node.  A
266         * "status" field in each node keeps track of whether a thread
267         * should block.  A node is signalled when its predecessor
268         * releases.  Each node of the queue otherwise serves as a
269         * specific-notification-style monitor holding a single waiting
270         * thread. The status field does NOT control whether threads are
271         * granted locks etc though.  A thread may try to acquire if it is
272         * first in the queue. But being first does not guarantee success;
273         * it only gives the right to contend.  So the currently released
274         * contender thread may need to rewait.
275         *
276         * <p>To enqueue into a CLH lock, you atomically splice it in as new
277         * tail. To dequeue, you just set the head field.  
278         * <pre>
279         *      +------+  prev +-----+       +-----+
280         * head |      | <---- |     | <---- |     |  tail
281         *      +------+       +-----+       +-----+
282         * </pre>
283         *
284         * <p>Insertion into a CLH queue requires only a single atomic
285         * operation on "tail", so there is a simple atomic point of
286         * demarcation from unqueued to queued. Similarly, dequeing
287         * involves only updating the "head". However, it takes a bit
288         * more work for nodes to determine who their successors are,
289         * in part to deal with possible cancellation due to timeouts
290         * and interrupts.
291         *
292         * <p>The "prev" links (not used in original CLH locks), are mainly
293         * needed to handle cancellation. If a node is cancelled, its
294         * successor is (normally) relinked to a non-cancelled
295         * predecessor. For explanation of similar mechanics in the case
296         * of spin locks, see the papers by Scott and Scherer at
297         * http://www.cs.rochester.edu/u/scott/synchronization/
298         * 
299         * <p>We also use "next" links to implement blocking mechanics.
300         * The thread id for each node is kept in its own node, so a
301         * predecessor signals the next node to wake up by traversing
302         * next link to determine which thread it is.  Determination of
303         * successor must avoid races with newly queued nodes to set
304         * the "next" fields of their predecessors.  This is solved
305         * when necessary by checking backwards from the atomically
306         * updated "tail" when a node's successor appears to be null.
307         * (Or, said differently, the next-links are an optimization
308         * so that we don't usually need a backward scan.)
309         *
310         * <p>Cancellation introduces some conservatism to the basic
311         * algorithms.  Since we must poll for cancellation of other
312         * nodes, we can miss noticing whether a cancelled node is
313         * ahead or behind us. This is dealt with by always unparking
314         * successors upon cancellation, allowing them to stabilize on
315         * a new predecessor.
316         *
317         * <p>CLH queues need a dummy header node to get started. But
318         * we don't create them on construction, because it would be wasted
319         * effort if there is never contention. Instead, the node
320         * is constructed and head and tail pointers are set upon first
321         * contention.
322         *
323         * <p>Threads waiting on Conditions use the same nodes, but
324         * use an additional link. Conditions only need to link nodes
325         * in simple (non-concurrent) linked queues because they are
326         * only accessed when exclusively held.  Upon await, a node is
327         * inserted into a condition queue.  Upon signal, the node is
328         * transferred to the main queue.  A special value of status
329         * field is used to mark which queue a node is on.
330         *
331         * <p>Thanks go to Dave Dice, Mark Moir, Victor Luchangco, Bill
332         * Scherer and Michael Scott, along with members of JSR-166
333         * expert group, for helpful ideas, discussions, and critiques
334         * on the design of this class.
335         */
336        static final class Node {
337            /** waitStatus value to indicate thread has cancelled */
338            static final int CANCELLED =  1;
339            /** waitStatus value to indicate thread needs unparking */
340            static final int SIGNAL    = -1;
341            /** waitStatus value to indicate thread is waiting on condition */
342            static final int CONDITION = -2;
343            /** Marker to indicate a node is waiting in shared mode */
344            static final Node SHARED = new Node();
345            /** Marker to indicate a node is waiting in exclusive mode */
346            static final Node EXCLUSIVE = null;
347    
348            /**
349             * Status field, taking on only the values:
350             *   SIGNAL:     The successor of this node is (or will soon be) 
351             *               blocked (via park), so the current node must 
352             *               unpark its successor when it releases or 
353             *               cancels. To avoid races, acquire methods must
354             *               first indicate they need a signal, 
355             *               then retry the atomic acquire, and then, 
356             *               on failure, block.
357             *   CANCELLED:  Node is cancelled due to timeout or interrupt
358             *               Nodes never leave this state. In particular,
359             *               a thread with cancelled node never again blocks.
360             *   CONDITION:  Node is currently on a condition queue
361             *               It will not be used as a sync queue node until
362             *               transferred. (Use of this value here
363             *               has nothing to do with the other uses
364             *               of the field, but simplifies mechanics.)
365             *   0:          None of the above
366             *
367             * The values are arranged numerically to simplify use.
368             * Non-negative values mean that a node doesn't need to
369             * signal. So, most code doesn't need to check for particular
370             * values, just for sign.
371             *
372             * The field is initialized to 0 for normal sync nodes, and
373             * CONDITION for condition nodes.  It is modified only using
374             * CAS.
375             */
376            volatile int waitStatus;
377    
378            /**
379             * Link to predecessor node that current node/thread relies on
380             * for checking waitStatus. Assigned during enqueing, and nulled
381             * out (for sake of GC) only upon dequeuing.  Also, upon
382             * cancellation of a predecessor, we short-circuit while
383             * finding a non-cancelled one, which will always exist
384             * because the head node is never cancelled: A node becomes
385             * head only as a result of successful acquire. A
386             * cancelled thread never succeeds in acquiring, and a thread only
387             * cancels itself, not any other node.
388             */
389            volatile Node prev;
390    
391            /**
392             * Link to the successor node that the current node/thread
393             * unparks upon release. Assigned once during enqueuing, and
394             * nulled out (for sake of GC) when no longer needed.  Upon
395             * cancellation, we cannot adjust this field, but can notice
396             * status and bypass the node if cancelled.  The enq operation
397             * does not assign next field of a predecessor until after
398             * attachment, so seeing a null next field does not
399             * necessarily mean that node is at end of queue. However, if
400             * a next field appears to be null, we can scan prev's from
401             * the tail to double-check.
402             */
403            volatile Node next;
404    
405            /**
406             * The thread that enqueued this node.  Initialized on
407             * construction and nulled out after use. 
408             */
409            volatile Thread thread;
410    
411            /** 
412             * Link to next node waiting on condition, or the special
413             * value SHARED.  Because condition queues are accessed only
414             * when holding in exclusive mode, we just need a simple
415             * linked queue to hold nodes while they are waiting on
416             * conditions. They are then transferred to the queue to
417             * re-acquire. And because conditions can only be exclusive,
418             * we save a field by using special value to indicate shared
419             * mode.
420             */
421            Node nextWaiter;
422    
423            /**
424             * Returns true if node is waiting in shared mode
425             */
426            final boolean isShared() {
427                return nextWaiter == SHARED;
428            }
429    
430            /**
431             * Returns previous node, or throws NullPointerException if
432             * null.  Use when predecessor cannot be null.
433             * @return the predecessor of this node
434             */
435            final Node predecessor() throws NullPointerException {
436                Node p = prev;
437                if (p == null)
438                    throw new NullPointerException(); 
439                else
440                    return p;
441            }
442    
443            Node() {    // Used to establish initial head or SHARED marker
444            }
445    
446            Node(Thread thread, Node mode) {     // Used by addWaiter
447                this.nextWaiter = mode;
448                this.thread = thread;
449            }
450    
451            Node(Thread thread, int waitStatus) { // Used by Condition
452                this.waitStatus = waitStatus;
453                this.thread = thread; 
454            }
455        }

457        /** 
458         * Head of the wait queue, lazily initialized.  Except for
459         * initialization, it is modified only via method setHead.  Note:
460         * If head exists, its waitStatus is guaranteed not to be
461         * CANCELLED.
462         */
463        private transient volatile Node head;
464    
465        /** 
466         * Tail of the wait queue, lazily initialized.  Modified only via
467         * method enq to add new wait node.
468         */
469        private transient volatile Node tail;

472         * The synchronization state.
473         */
474        private volatile int state;

494        /**
495         * Atomically sets synchronization state to the given updated
496         * value if the current state value equals the expected value.
497         * This operation has memory semantics of a <tt>volatile</tt> read
498         * and write.
499         * @param expect the expected value
500         * @param update the new value
501         * @return true if successful. False return indicates that

511        /**
512         * Insert node into queue, initializing if necessary. See picture above.
513         * @param node the node to insert
514         * @return node's predecessor
515         */
516        private Node enq(final Node node) {
517            for (;;) {
518                Node t = tail;
519                if (t == null) { // Must initialize
520                    Node h = new Node(); // Dummy header
521                    h.next = node;
522                    node.prev = h;
523                    if (compareAndSetHead(h)) {
524                        tail = node;
525                        return h;
526                    }
527                }
528                else {
529                    node.prev = t;     
530                    if (compareAndSetTail(t, node)) {
531                        t.next = node; 
532                        return t; 
533                    }
534                }
535            }
536        }
537    
538        /**
539         * Create and enq node for given thread and mode
540         * @param current the thread
541         * @param mode Node.EXCLUSIVE for exclusive, Node.SHARED for shared
542         * @return the new node
543         */
544        private Node addWaiter(Node mode) {
545            Node node = new Node(Thread.currentThread(), mode);
546            // Try the fast path of enq; backup to full enq on failure
547            Node pred = tail;
548            if (pred != null) {
549                node.prev = pred;     
550                if (compareAndSetTail(pred, node)) {
551                    pred.next = node; 
552                    return node;
553                }
554            }
555            enq(node);
556            return node;
557        }

559        /**
560         * Set head of queue to be node, thus dequeuing. Called only by
561         * acquire methods.  Also nulls out unused fields for sake of GC
562         * and to suppress unnecessary signals and traversals.
563         * @param node the node 
564         */
565        private void setHead(Node node) {
566            head = node;
567            node.thread = null;
568            node.prev = null; 
569        }

571        /**
572         * Wake up node's successor, if one exists.
573         * @param node the node
574         */
575        private void unparkSuccessor(Node node) {
576            /*
577             * Try to clear status in anticipation of signalling.  It is
578             * OK if this fails or if status is changed by waiting thread.
579             */
580            compareAndSetWaitStatus(node, Node.SIGNAL, 0);
581            
582            /*
583             * Thread to unpark is held in successor, which is normally
584             * just the next node.  But if cancelled or apparently null,
585             * traverse backwards from tail to find the actual
586             * non-cancelled successor.
587             */
588            Thread thread;
589            Node s = node.next;
590            if (s != null && s.waitStatus <= 0)
591                thread = s.thread;
592            else {
593                thread = null;
594                for (s = tail; s != null && s != node; s = s.prev) 
595                    if (s.waitStatus <= 0)
596                        thread = s.thread;
597            }
598            LockSupport.unpark(thread);
599        }
600    
601        /**
602         * Set head of queue, and check if successor may be waiting
603         * in shared mode, if so propagating if propagate > 0.
604         * @param pred the node holding waitStatus for node
605         * @param node the node 
606         * @param propagate the return value from a tryAcquireShared
607         */
608        private void setHeadAndPropagate(Node node, int propagate) {
609            setHead(node);
610            if (propagate > 0 && node.waitStatus != 0) {
611                /*
612                 * Don't bother fully figuring out successor.  If it
613                 * looks null, call unparkSuccessor anyway to be safe.
614                 */
615                Node s = node.next; 
616                if (s == null || s.isShared())
617                    unparkSuccessor(node);
618            }
619        }

623        /**
624         * Cancel an ongoing attempt to acquire.
625         * @param node the node
626         */
627        private void cancelAcquire(Node node) {
628            if (node != null) { // Ignore if node doesn't exist
629                node.thread = null;
630                // Can use unconditional write instead of CAS here
631                node.waitStatus = Node.CANCELLED;
632                unparkSuccessor(node);
633            }

636        /**
637         * Checks and updates status for a node that failed to acquire.
638         * Returns true if thread should block. This is the main signal
639         * control in all acquire loops.  Requires that pred == node.prev
640         * @param pred node's predecessor holding status
641         * @param node the node 
642         * @return true if thread should block
643         */
644        private static boolean shouldParkAfterFailedAcquire(Node pred, Node node) {
645            int s = pred.waitStatus;
646            if (s < 0)
647                /*
648                 * This node has already set status asking a release
649                 * to signal it, so it can safely park
650                 */
651                return true;
652            if (s > 0) 
653                /*
654                 * Predecessor was cancelled. Move up to its predecessor
655                 * and indicate retry.
656                 */
657                node.prev = pred.prev;
658            else
659                /*
660                 * Indicate that we need a signal, but don't park yet. Caller
661                 * will need to retry to make sure it cannot acquire before
662                 * parking.
663                 */
664                compareAndSetWaitStatus(pred, 0, Node.SIGNAL);
665            return false;
666        }

693        /**
694         * Acquire in exclusive uninterruptible mode for thread already in
695         * queue. Used by condition wait methods as well as acquire.
696         * @param node the node
697         * @param arg the acquire argument
698         * @return true if interrupted while waiting
699         */
700        final boolean acquireQueued(final Node node, int arg) {
701            try {
702                boolean interrupted = false;
703                for (;;) {
704                    final Node p = node.predecessor();
705                    if (p == head && tryAcquire(arg)) {
706                        setHead(node);
707                        p.next = null; // help GC
708                        return interrupted;
709                    }
710                    if (shouldParkAfterFailedAcquire(p, node) && 
711                        parkAndCheckInterrupt()) 
712                        interrupted = true;
713                }
714            } catch (RuntimeException ex) {
715                cancelAcquire(node);
716                throw ex;
717            }
718        }

751         * @param nanosTimeout max wait time
752         * @return true if acquired
753         */
754        private boolean doAcquireNanos(int arg, long nanosTimeout) 
755            throws InterruptedException {
756            long lastTime = System.nanoTime();
757            final Node node = addWaiter(Node.EXCLUSIVE);
758            try {
759                for (;;) {
760                    final Node p = node.predecessor();
761                    if (p == head && tryAcquire(arg)) {
762                        setHead(node);
763                        p.next = null; // help GC
764                        return true;
765                    }

771                        LockSupport.parkNanos(nanosTimeout);

778                }
779            } catch (RuntimeException ex) {
780                cancelAcquire(node);
781                throw ex;
782            }
783            // Arrive here only if interrupted
784            cancelAcquire(node);
785            throw new InterruptedException();
786        }

1235        /**
1236         * Version of getFirstQueuedThread called when fastpath fails
1237         */
1238        private Thread fullGetFirstQueuedThread() {
1239            /*
1240             * This loops only if the queue changes while we read sets of
1241             * fields.
1242             */
1243            for (;;) {
1244                Node h = head;
1245                if (h == null)                    // No queue
1246                    return null;
1247    
1248                /*
1249                 * The first node is normally h.next. Try to get its
1250                 * thread field, ensuring consistent reads: If thread
1251                 * field is nulled out or s.prev is no longer head, then
1252                 * some other thread(s) concurrently performed setHead in
1253                 * between some of our reads, so we must reread.
1254                 */
1255                Node s = h.next;
1256                if (s != null) {
1257                    Thread st = s.thread;
1258                    Node sp = s.prev;
1259                    if (st != null && sp == head)
1260                        return st;
1261                }
1262    
1263                /*
1264                 * Head's next field might not have been set yet, or may
1265                 * have been unset after setHead. So we must check to see
1266                 * if tail is actually first node, in almost the same way
1267                 * as above.
1268                 */
1269                Node t = tail; 
1270                if (t == h)                       // Empty queue
1271                    return null;
1272    
1273                if (t != null) {
1274                    Thread tt = t.thread;
1275                    Node tp = t.prev;
1276                    if (tt != null && tp == head)
1277                        return tt;
1278                }
1279            }
1280        }

1399        /**
1400         * Returns true if a node, always one that was initially placed on
1401         * a condition queue, is now waiting to reacquire on sync queue.
1402         * @param node the node
1403         * @return true if is reacquiring
1404         */
1405        final boolean isOnSyncQueue(Node node) {
1406            if (node.waitStatus == Node.CONDITION || node.prev == null)
1407                return false;
1408            if (node.next != null) // If has successor, it must be on queue
1409                return true;
1410            /*
1411             * node.prev can be non-null, but not yet on queue because
1412             * the CAS to place it on queue can fail. So we have to
1413             * traverse from tail to make sure it actually made it.  It
1414             * will always be near the tail in calls to this method, and
1415             * unless the CAS failed (which is unlikely), it will be
1416             * there, so we hardly ever traverse much.
1417             */
1418            return findNodeFromTail(node);
1419        }

1421        /**
1422         * Returns true if node is on sync queue by searching backwards from tail.
1423         * Called only when needed by isOnSyncQueue.
1424         * @return true if present
1425         */
1426        private boolean findNodeFromTail(Node node) {
1427            Node t = tail; 
1428            for (;;) {
1429                if (t == node)
1430                    return true;
1431                if (t == null)
1432                    return false;
1433                t = t.prev;
1434            }
1435        }

1437        /**
1438         * Transfers a node from a condition queue onto sync queue. 
1439         * Returns true if successful.
1440         * @param node the node
1441         * @return true if successfully transferred (else the node was
1442         * cancelled before signal).
1443         */
1444        final boolean transferForSignal(Node node) {
1445            /*
1446             * If cannot change waitStatus, the node has been cancelled.
1447             */
1448            if (!compareAndSetWaitStatus(node, Node.CONDITION, 0))
1449                return false;
1450    
1451            /*
1452             * Splice onto queue and try to set waitStatus of predecessor to
1453             * indicate that thread is (probably) waiting. If cancelled or
1454             * attempt to set waitStatus fails, wake up to resync (in which
1455             * case the waitStatus can be transiently and harmlessly wrong).
1456             */
1457            Node p = enq(node);
1458            int c = p.waitStatus;
1459            if (c > 0 || !compareAndSetWaitStatus(p, c, Node.SIGNAL)) 
1460                LockSupport.unpark(node.thread);
1461            return true;
1462        }

1464        /**
1465         * Transfers node, if necessary, to sync queue after a cancelled
1466         * wait. Returns true if thread was cancelled before being
1467         * signalled.
1468         * @param current the waiting thread
1469         * @param node its node
1470         * @return true if cancelled before the node was signalled.
1471         */
1472        final boolean transferAfterCancelledWait(Node node) {
1473            if (compareAndSetWaitStatus(node, Node.CONDITION, 0)) {
1474                enq(node);
1475                return true;
1476            }
1477            /*
1478             * If we lost out to a signal(), then we can't proceed
1479             * until it finishes its enq().  Cancelling during an
1480             * incomplete transfer is both rare and transient, so just
1481             * spin.
1482             */
1483            while (!isOnSyncQueue(node)) 
1484                Thread.yield();
1485            return false;
1486        }

1488        /**
1489         * Invoke release with current state value; return saved state.
1490         * Cancel node and throw exception on failure.
1491         * @param node the condition node for this wait
1492         * @return previous sync state
1493         */
1494        final int fullyRelease(Node node) {
1495            try {
1496                int savedState = getState();
1497                if (release(savedState))
1498                    return savedState;
1499            } catch(RuntimeException ex) {
1500                node.waitStatus = Node.CANCELLED;
1501                throw ex;
1502            }
1503            // reach here if release fails
1504            node.waitStatus = Node.CANCELLED;
1505            throw new IllegalMonitorStateException();
1506        }

1692             * <ol>
1693             * <li> Save lock state returned by {@link #getState} 
1694             * <li> Invoke {@link #release} with 
1695             *      saved state as argument, throwing 
1696             *      IllegalMonitorStateException  if it fails.
1697             * <li> Block until signalled
1698             * <li> Reacquire by invoking specialized version of
1699             *      {@link #acquire} with saved state as argument.
1700             * </ol>

1703                Node node = addConditionWaiter();
1704                int savedState = fullyRelease(node);
1705                boolean interrupted = false;
1706                while (!isOnSyncQueue(node)) {
1707                    LockSupport.park();
1708                    if (Thread.interrupted())

1716             * For interruptible waits, we need to track whether to throw
1717             * InterruptedException, if interrupted while blocked on
1718             * condition, versus reinterrupt current thread, if
1719             * interrupted while blocked waiting to re-acquire.
1720             */

1722            /** Mode meaning to reinterrupt on exit from wait */
1723            private static final int REINTERRUPT =  1;
1724            /** Mode meaning to throw InterruptedException on exit from wait */
1725            private static final int THROW_IE    = -1;

1729             * before signalled, REINTERRUPT if after signalled, or
1730             * 0 if not interrupted.
1731             */

1733                return (Thread.interrupted()) ?
1734                    ((transferAfterCancelledWait(node))? THROW_IE : REINTERRUPT) :
1735                    0;
1736            }

1740             * do nothing, depending on mode.
1741             */

1743                throws InterruptedException {
1744                if (interruptMode == THROW_IE)
1745                    throw new InterruptedException();
1746                else if (interruptMode == REINTERRUPT)
1747                    Thread.currentThread().interrupt();

1750            /**
1751             * Implements interruptible condition wait.
1752             * <ol>
1753             * <li> If current thread is interrupted, throw InterruptedException
1754             * <li> Save lock state returned by {@link #getState} 
1755             * <li> Invoke {@link #release} with 
1756             *      saved state as argument, throwing 
1757             *      IllegalMonitorStateException  if it fails.
1758             * <li> Block until signalled or interrupted
1759             * <li> Reacquire by invoking specialized version of
1760             *      {@link #acquire} with saved state as argument.
1761             * <li> If interrupted while blocked in step 4, throw exception
1762             * </ol>
1763             * 
1764             * @throws InterruptedException if the current thread is interrupted (and
1765             * interruption of thread suspension is supported).
1766             */
1767            public final void await() throws InterruptedException {
1768                if (Thread.interrupted()) 
1769                    throw new InterruptedException();
1770                Node node = addConditionWaiter();
1771                int savedState = fullyRelease(node);
1772                int interruptMode = 0;
1773                while (!isOnSyncQueue(node)) {
1774                    LockSupport.park();
1775                    if ((interruptMode = checkInterruptWhileWaiting(node)) != 0)
1776                        break;
1777                }
1778                if (acquireQueued(node, savedState) && interruptMode != THROW_IE)
1779                    interruptMode = REINTERRUPT;
1780                if (interruptMode != 0)
1781                    reportInterruptAfterWait(interruptMode);
1782            }

1785             * Implements timed condition wait.
1786             * <ol>
1787             * <li> If current thread is interrupted, throw InterruptedException
1788             * <li> Save lock state returned by {@link #getState} 
1789             * <li> Invoke {@link #release} with 
1790             *      saved state as argument, throwing 
1791             *      IllegalMonitorStateException  if it fails.
1792             * <li> Block until signalled, interrupted, or timed out
1793             * <li> Reacquire by invoking specialized version of
1794             *      {@link #acquire} with saved state as argument.
1795             * <li> If interrupted while blocked in step 4, throw InterruptedException
1796             * </ol>
1797             * 
1798             * @param nanosTimeout the maximum time to wait, in nanoseconds
1799             * @return A value less than or equal to zero if the wait has
1800             * timed out; otherwise an estimate, that
1801             * is strictly less than the <tt>nanosTimeout</tt> argument,
1802             * of the time still remaining when this method returned.
1803             *
1804             * @throws InterruptedException if the current thread is interrupted (and
1805             * interruption of thread suspension is supported).
1806             */
1807            public final long awaitNanos(long nanosTimeout) throws InterruptedException {
1808                if (Thread.interrupted()) 
1809                    throw new InterruptedException();
1810                Node node = addConditionWaiter();
1811                int savedState = fullyRelease(node);
1812                long lastTime = System.nanoTime();
1813                int interruptMode = 0;
1814                while (!isOnSyncQueue(node)) {
1815                    if (nanosTimeout <= 0L) {
1816                        transferAfterCancelledWait(node); 
1817                        break;
1818                    }
1819                    LockSupport.parkNanos(nanosTimeout);
1820                    if ((interruptMode = checkInterruptWhileWaiting(node)) != 0)
1821                        break;
1822                    long now = System.nanoTime();
1823                    nanosTimeout -= now - lastTime;
1824                    lastTime = now;
1825                }
1826                if (acquireQueued(node, savedState) && interruptMode != THROW_IE)
1827                    interruptMode = REINTERRUPT;
1828                if (interruptMode != 0)
1829                    reportInterruptAfterWait(interruptMode);
1830                return nanosTimeout - (System.nanoTime() - lastTime);

1987             */
1988            protected final Collection<Thread> getWaitingThreads() {
1989                if (!isHeldExclusively()) 
1990                    throw new IllegalMonitorStateException();
1991                ArrayList<Thread> list = new ArrayList<Thread>();
1992                for (Node w = firstWaiter; w != null; w = w.nextWaiter) {
1993                    if (w.waitStatus == Node.CONDITION) {
1994                        Thread t = w.thread;
1995                        if (t != null)
1996                            list.add(t);
1997                    }
1998                }
1999                return list;
2000            }
2001        }
