020    /**
021     * An interface which provides comprehensive information about the database
022     * management system and its supported features.
023     * <p>
024     * This interface is implemented by JDBC driver vendors in order to provide
025     * information about the underlying database capabilities in association with
026     * the JDBC driver.
027     * </p>
028     * <p>
029     * Some of the methods in this interface take string parameters which are
030     * patterns. Within these string patterns, {@code '%'} and {@code '_'}
031     * characters have special meanings. {@code '%'} means
032     * "match any substring of 0 or more characters". {@code '_'} means
033     * "match any character". Only metadata entries that match the pattern are
034     * returned. If such a search pattern string is set to {@code null}, that
035     * argument's criteria are dropped from the search.
036     * </p>
037     *  
038     * @since Android 1.0

469        /**
470         * Returns a {@code ResultSet} describing a subset of the attributes of a 
471         * specified SQL User Defined Type (UDT) for a specified schema and catalog.
472         * The subset is determined by restricting to those attributes whose 
473         * name matches the {@code attributeNamePattern} and whose type name 
474         * matches the {@code typeNamePattern}. Each row of the {@code ResultSet}
475         * describes one attribute, and the rows are ordered by the columns {@code TYPE_SCHEM}, 
476         * {@code TYPE_NAME} and {@code ORDINAL_POSITION}. Inherited attributes
477         * are not included.
478         * <p>
479         * The columns of the returned {@code ResultSet} object have the following
480         * names and meanings:
481         * <ol>
482         * <li>{@code TYPE_CAT} - String - the type catalog name (possibly {@code
483         * null})</li>
484         * <li>{@code TYPE_SCHEM} - String - the type schema name (possibly {@code
485         * null})</li>
486         * <li>{@code TYPE_NAME} - String - the type name</li>
487         * <li>{@code ATTR_NAME} - String - the attribute name</li>
488         * <li>{@code DATA_TYPE} - int - the attribute type as defined in {@code
489         * java.sql.Types}</li>
490         * <li>{@code ATTR_TYPE_NAME} - String - the attribute type name. This
491         * depends on the data source. For a {@code UDT} the name is fully
492         * qualified. For a {@code REF} it is both fully qualified and represents
493         * the target type of the reference.</li>
494         * <li>{@code ATTR_SIZE} - int - the column size. When referring to char and
495         * date types this value is the maximum number of characters. When referring
496         * to numeric types is is the precision.</li>
497         * <li>{@code DECIMAL_DIGITS} - int - how many fractional digits are
498         * supported</li>
499         * <li>{@code NUM_PREC_RADIX} - int - numeric values radix</li>
500         * <li>{@code NULLABLE} - int - whether {@code NULL} is permitted:
501         * <ul>
502         * <li>DatabaseMetaData.attributeNoNulls - {@code NULL} values not permitted</li>
503         * <li>DatabaseMetaData.attributeNullable - {@code NULL} values definitely
504         * permitted</li>
505         * <li>DatabaseMetaData.attributeNullableUnknown - unknown</li>
506         * </ul>
507         * </li>
508         * <li>{@code REMARKS} - String - a comment describing the attribute
509         * (possibly {@code null})</li>
510         * <li>ATTR_DEF - String - Default value for the attribute (possibly {@code
511         * null})</li>
512         * <li>{@code SQL_DATA_TYPE} - int - not used</li>
513         * <li>SQL_DATETIME_SUB - int - not used</li>
514         * <li>CHAR_OCTET_LENGTH - int - for {@code CHAR} types, the max number of
515         * bytes in the column</li>
516         * <li>ORDINAL_POSITION - int - The index of the column in the table (where
517         * the count starts from 1, not 0)</li>
518         * <li>IS_NULLABLE - String - {@code "NO"} = the column does not allow {@code
519         * NULL}s, {@code "YES"} = the column allows {@code NULL}s, "" = status unknown</li>
520         * <li>{@code SCOPE_CATALOG} - String - if the {@code DATA_TYPE} is {@code REF}, 
521         * this gives the catalog of the table corresponding to the attribute's scope. 
522         * NULL if the {@code DATA_TYPE} is not REF.</li>
523         * <li>{@code SCOPE_SCHEMA} - String - if the {@code DATA_TYPE} is {@code REF}, 
524         * this gives the schema of the table corresponding to the attribute's scope. 
525         * NULL if the {@code DATA_TYPE} is not REF.</li>
526         * <li>{@code SCOPE_TABLE} - String - if the {@code DATA_TYPE} is {@code REF}, 
527         * this gives the name of the table corresponding to the attribute's scope. 
528         * NULL if the {@code DATA_TYPE} is not REF.</li>
529         * <li>{@code SOURCE_DATA_TYPE} - String - The source type for a user
530         * generated REF type or for a Distinct type. ({@code NULL} if {@code
531         * DATA_TYPE} is not DISTINCT or a user generated REF)</li>
532         * </ol>
533         * </p>
534         * 
535         * @param catalog
536         *            a catalog name. {@code null} is used to imply no narrowing of
537         *            the search by catalog name. Otherwise, the name must match a
538         *            catalog name held in the database, with "" used to retrieve
539         *            those without a catalog name.
540         * @param schemaPattern
541         *            a schema name pattern. {@code null} is used to imply no
542         *            narrowing of the search by a schema name. Otherwise, the name
543         *            must match a schema name in the database, with "" used to
544         *            retrieve those without a schema name.
545         * @param typeNamePattern
546         *            a type name. This pattern must match the type name stored in
547         *            the database.
548         * @param attributeNamePattern
549         *            an Attribute name. This pattern must match the attribute name as stored in
550         *            the database.
551         * @return a {@code ResultSet}, where each row is an attribute description.
552         * @throws SQLException
553         *             if there is a database error.
554         * @since Android 1.0
555         */
556        public ResultSet getAttributes(String catalog, String schemaPattern,
557                String typeNamePattern, String attributeNamePattern)
558                throws SQLException;

560        /**
561         * Returns a list of a table's optimal set of columns that uniquely
562         * identify the rows. The results are ordered by {@code SCOPE} (see below).
563         * <p>
564         * The results are returned as a table, with one entry for each column, as
565         * follows:
566         * <ol>
567         * <li>{@code SCOPE} - short - the {@code SCOPE} of the result, as follows:
568         * <ul>
569         * <li>{@code DatabaseMetaData.bestRowTemporary} - the result is very temporary, 
570         * only valid while on the current row</li>
571         * <li>{@code DatabaseMetaData.bestRowTransaction} - the result is good for remainder of
572         * current transaction</li>
573         * <li>{@code DatabaseMetaData.bestRowSession} - the result is good for remainder of
574         * database session</li>
575         * </ul>
576         * </li>
577         * <li>{@code COLUMN_NAME} - String - the column name</li>
578         * <li>{@code DATA_TYPE} - int - the Type of the data, as defined in {@code
579         * java.sql.Types}</li>
580         * <li>{@code TYPE_NAME} - String - the Name of the type - database dependent.
581         * For UDT types the name is fully qualified</li>
582         * <li>{@code COLUMN_SIZE} - int - the precision of the data in the column</li>
583         * <li>{@code BUFFER_LENGTH} - int - not used</li>
584         * <li>{@code DECIMAL_DIGITS} - short - number of fractional digits</li>
585         * <li>{@code PSEUDO_COLUMN} - short - whether this is a pseudo column (e.g.
586         * an Oracle {@code ROWID}):
587         * <ul>
588         * <li>{@code DatabaseMetaData.bestRowUnknown} - it is not known whether this is
589         * a pseudo column</li>
590         * <li>{@code DatabaseMetaData.bestRowNotPseudo} - the column is not pseudo</li>
591         * <li>{@code DatabaseMetaData.bestRowPseudo} - the column is a pseudo column</li>
592         * </ul>
593         * </li>
594         * </ol>
595         * </p>
596         * 
597         * @param catalog
598         *            a catalog name. {@code null} is used to imply no narrowing of
599         *            the search by catalog name. Otherwise, the name must match a
600         *            catalog name held in the database, with "" used to retrieve
601         *            those without a catalog name.
602         * @param schema
603         *            a schema name pattern. {@code null} is used to imply no
604         *            narrowing of the search by schema name. Otherwise, the name
605         *            must match a schema name in the database, with "" used to
606         *            retrieve those without a schema name.
607         * @param table
608         *            the table name. This must match the name of the table as
609         *            declared in the database.
610         * @param scope
611         *            the {@code SCOPE} of interest, values as defined above.
612         * @param nullable
613         *            {@code true} = include columns that are nullable, {@code
614         *            false} = do not include nullable columns.
615         * @return a {@code ResultSet} where each row is a description of a column
616         *         and the complete set of rows is the optimal set for this table.
617         * @throws SQLException
618         *             if there is a database error.
619         * @since Android 1.0
620         */
621        public ResultSet getBestRowIdentifier(String catalog, String schema,
622                String table, int scope, boolean nullable) throws SQLException;

664         * <ol>
665         * <li>{@code TABLE_CAT} - String - the catalog name (possibly {@code null})</li>
666         * <li>{@code TABLE_SCHEM} - String - the schema name (possibly {@code null})</li>

668         * <li>{@code COLUMN_NAME} - String - the Column name</li>
669         * <li>{@code GRANTOR} - String - the grantor of access (possibly {@code
670         * null})</li>
671         * <li>{@code PRIVILEGE} - String - Access right - one of SELECT, INSERT,
672         * UPDATE, REFERENCES,...</li>
673         * <li>{@code IS_GRANTABLE} - String - {@code "YES"} implies that the
674         * receiver can grant access to others, {@code "NO"} if the receiver cannot
675         * grant access to others, {@code null} if unknown.</li>
676         * </ol>
677         * </p>
678         * 
679         * @param catalog
680         *            a catalog name. {@code null} is used to imply no narrowing of
681         *            the search by catalog name. Otherwise, the name must match a
682         *            catalog name held in the database, with "" used to retrieve
683         *            those without a catalog name.
684         * @param schema
685         *            a schema name pattern. {@code null} is used to imply no
