023    import java.nio.channels.spi.AbstractInterruptibleChannel;

025    /**
026     * An abstract channel type for interaction with a platform file.
027     * <p>
028     * A {@code FileChannel} defines the methods for reading, writing, memory
029     * mapping, and manipulating the logical state of a platform file. This type
030     * does not have a method for opening files, since this behavior has been
031     * delegated to the {@link java.io.FileInputStream},
032     * {@link java.io.FileOutputStream} and {@link java.io.RandomAccessFile} types.
033     * </p>
034     * <p>
035     * FileChannels created from a {@code FileInputStream} or a
036     * {@code RandomAccessFile} created in mode "r", are read-only. FileChannels
037     * created from a {@code FileOutputStream} are write-only. FileChannels created
038     * from a {@code RandomAccessFile} created in mode "rw" are read/write.
039     * FileChannels created from a {@code RandomAccessFile} that was opened in
040     * append-mode will also be in append-mode -- meaning that each write will be
041     * proceeded by a seek to the end of file.
042     * </p>
043     * <p>
044     * FileChannels have a virtual pointer into the file which is referred to as a
045     * file <em>position</em>. The position can be manipulated by moving it
046     * within the file, and the current position can be queried.
047     * </p>
048     * <p>
049     * FileChannels also have an associated <em>size</em>. The size of the file
050     * is the number of bytes that it currently contains. The size can be
051     * manipulated by adding more bytes to the end of the file (which increases the
052     * size) or truncating the file (which decreases the size). The current size can
053     * also be queried.
054     * </p>
055     * <p>
056     * FileChannels have operations beyond the simple read, write, and close. They
057     * can also:
058     * <ul>
059     * <li>request that cached data be forced onto the disk,</li>
060     * <li>lock ranges of bytes associated with the file,</li>
061     * <li>transfer data directly to another channel in a manner that has the
062     * potential to be optimized by the platform,</li>
063     * <li>memory-mapping files into NIO buffers to provide efficient manipulation
064     * of file data,</li>
065     * <li>read and write to the file at absolute byte offsets in a fashion that
066     * does not modify the current position.</li>
067     * </ul>
068     * </p>
069     * <p>
070     * FileChannels are thread-safe. Only one operation involving manipulation of
071     * the file position may be executed at the same time. Subsequent calls to such
072     * operations will block, and one of those blocked will be freed to continue
073     * when the first operation has completed. There is no ordered queue or fairness
074     * applied to the blocked threads.
075     * </p>
076     * <p>
077     * It is undefined whether operations that do not manipulate the file position
078     * will also block when there are any other operations in-flight.
079     * </p>
080     * <p>
081     * The logical view of the underlying file is consistent across all FileChannels
082     * and I/O streams opened on the same file by the same virtual machine process.
083     * Therefore, modifications performed via a channel will be visible to the
084     * stream and vice versa; this includes modifications to the file position,
085     * content, size, etc.
086     * </p>
087     * 
088     * @since Android 1.0
089     */
090    public abstract class FileChannel extends AbstractInterruptibleChannel
091            implements GatheringByteChannel, ScatteringByteChannel, ByteChannel {

151        /**
152         * Requests that all updates to this channel are committed to the storage
153         * device.
154         * <p>
155         * When this method returns, all modifications made to the platform file
156         * underlying this channel have been committed if the file resides on a
157         * local storage device. If the file is not hosted locally, for example on a
158         * networked file system, then applications cannot be certain that the
159         * modifications have been committed.
160         * </p>
161         * <p>
162         * There are no assurances given that changes made to the file using methods
163         * defined elsewhere will be committed. For example, changes made via a
164         * mapped byte buffer may not be committed.
165         * </p>
166         * <p>
167         * The <code>metadata</code> parameter indicates whether the update should
168         * include the file's metadata such as last modification time, last access
169         * time, etc. Note that passing <code>true</code> may invoke an underlying
170         * write to the operating system (if the platform is maintaining metadata
171         * such as last access time), even if the channel is opened read-only.
172         * 
173         * @param metadata
174         *            {@code true} if the file metadata should be flushed in
175         *            addition to the file content, {@code false} otherwise.
176         * @throws ClosedChannelException
177         *             if this channel is already closed.
178         * @throws IOException
179         *             if another I/O error occurs.
180         * @since Android 1.0
181         */
182        public abstract void force(boolean metadata) throws IOException;

184        /**
185         * Obtains an exclusive lock on this file.
186         * <p>
187         * This is a convenience method for acquiring a maximum length lock on a
188         * file. It is equivalent to:
189         * {@code fileChannel.lock(0L, Long.MAX_VALUE, false);}
190         * 
191         * @return the lock object representing the locked file area.
192         * @throws ClosedChannelException
193         *             the file channel is closed.
194         * @throws NonWritableChannelException
195         *             this channel was not opened for writing.
196         * @throws OverlappingFileLockException
197         *             either a lock is already held that overlaps this lock
198         *             request, or another thread is waiting to acquire a lock that
199         *             will overlap with this request.
200         * @throws FileLockInterruptionException
201         *             the calling thread was interrupted while waiting to acquire
202         *             the lock.
203         * @throws AsynchronousCloseException
204         *             the channel was closed while the calling thread was waiting
205         *             to acquire the lock.
206         * @throws IOException
207         *             if another I/O error occurs while obtaining the requested
208         *             lock.
209         * @since Android 1.0
210         */
211        public final FileLock lock() throws IOException {
212            return lock(0L, Long.MAX_VALUE, false);
213        }

215        /**
216         * Obtains a lock on a specified region of the file.
217         * <p>
218         * This is the blocking version of lock acquisition, see also the
219         * <code>tryLock()</code> methods.
220         * </p>
221         * <p>
222         * Attempts to acquire an overlapping lock region will fail. The attempt
223         * will fail if the overlapping lock has already been obtained, or if
224         * another thread is currently waiting to acquire the overlapping lock.
225         * </p>
226         * <p>
227         * If the request is not for an overlapping lock, the thread calling this
228         * method will block until the lock is obtained (likely by no contention or
229         * another process releasing a lock), or until this thread is interrupted or
230         * the channel is closed.
231         * </p>
232         * <p>
233         * If the lock is obtained successfully then the {@link FileLock} object
234         * returned represents the lock for subsequent operations on the locked
235         * region.
236         * </p>
237         * <p>
238         * If the thread is interrupted while waiting for the lock, the thread is
239         * set to the interrupted state and throws a
240         * {@link FileLockInterruptionException}. If this channel is closed while
241         * the thread is waiting to obtain the lock then the thread throws a
242         * {@link AsynchronousCloseException}.
243         * </p>
244         * <p>
245         * There is no requirement for the position and size to be within the
246         * current start and length of the file.
247         * </p>
248         * <p>
249         * Some platforms do not support shared locks, and if a request is made for
250         * a shared lock on such a platform, this method will attempt to acquire an
251         * exclusive lock instead. It is undefined whether the lock obtained is
252         * advisory or mandatory.
253         * </p>
254         * 
255         * @param position
256         *            the starting position for the locked region.
257         * @param size
258         *            the length of the locked region in bytes.
259         * @param shared
260         *            a flag indicating whether an attempt should be made to acquire
261         *            a shared lock.
262         * @return the file lock object.
263         * @throws IllegalArgumentException
264         *             if {@code position} or {@code size} is negative.
265         * @throws ClosedChannelException
266         *             if this channel is closed.
267         * @throws OverlappingFileLockException
268         *             if the requested region overlaps an existing lock or pending
269         *             lock request.
270         * @throws NonReadableChannelException
271         *             if the channel is not opened in read-mode but shared is true.
272         * @throws NonWritableChannelException
273         *             if the channel is not opened in write mode but shared is
274         *             false.
275         * @throws AsynchronousCloseException
276         *             if this channel is closed by another thread while this method
277         *             is executing.
278         * @throws FileLockInterruptionException
279         *             if the thread is interrupted while in the state of waiting on
280         *             the desired file lock.
281         * @throws IOException
282         *             if another I/O error occurs.
283         * @since Android 1.0
284         */
285        public abstract FileLock lock(long position, long size, boolean shared)
286                throws IOException;

289         * Maps the file into memory. There can be three modes: read-only,
290         * read/write and private. After mapping, changes made to memory or the file
291         * channel do not affect the other storage place.

303         * @throws NonReadableChannelException

307         *             if the FileChannel is not opened for writing but the given

309         * @throws IllegalArgumentException
310         *             if the given parameters of position and size are not correct.

334         * Sets the file position pointer to a new value.
335         * <p>

343         * @param offset
344         *            the new file position, in bytes.
345         * @return the receiver.
346         * @throws IllegalArgumentException
347         *             if the new position is negative.
348         * @throws ClosedChannelException
349         *             if this channel is closed.
350         * @throws IOException
351         *             if another I/O error occurs.
352         * @since Android 1.0

356        /**
357         * Reads bytes from this file channel into the given buffer.
358         * <p>
359         * The maximum number of bytes that will be read is the remaining number of
360         * bytes in the buffer when the method is invoked. The bytes will be copied
361         * into the buffer starting at the buffer's current position.
362         * </p>
363         * <p>
364         * The call may block if other threads are also attempting to read from this
365         * channel.
366         * </p>
367         * <p>
368         * Upon completion, the buffer's position is set to the end of the bytes
369         * that have been read. The buffer's limit is not changed.
370         * </p>
371         * 
372         * @param buffer
373         *            the byte buffer to receive the bytes.
374         * @return the number of bytes actually read.
375         * @throws AsynchronousCloseException
376         *             if another thread closes the channel during the read.
377         * @throws ClosedByInterruptException
378         *             if another thread interrupts the calling thread during the
379         *             read.
380         * @throws ClosedChannelException
381         *             if this channel is closed.
382         * @throws IOException
383         *             if another I/O error occurs, details are in the message.
384         * @throws NonReadableChannelException
385         *             if the channel has not been opened in a mode that permits
386         *             reading.
387         * @since Android 1.0
388         */
389        public abstract int read(ByteBuffer buffer) throws IOException;

391        /**
392         * Reads bytes from this file channel into the given buffer starting from
393         * the specified file position.
394         * <p>
395         * The bytes are read starting at the given file position (up to the
396         * remaining number of bytes in the buffer). The number of bytes actually
397         * read is returned.
398         * </p>
399         * <p>
400         * If {@code position} is beyond the current end of file, then no bytes are
401         * read.
402         * </p>
403         * <p>
404         * Note that the file position is unmodified by this method.
405         * </p>
406         * 
407         * @param buffer
408         *            the buffer to receive the bytes.
409         * @param position
410         *            the (non-negative) position at which to read the bytes.
411         * @return the number of bytes actually read.
412         * @throws AsynchronousCloseException
413         *             if this channel is closed by another thread while this method
414         *             is executing.
415         * @throws ClosedByInterruptException
416         *             if another thread interrupts the calling thread while this
417         *             operation is in progress. The calling thread will have the
418         *             interrupt state set, and the channel will be closed.
419         * @throws ClosedChannelException
420         *             if this channel is closed.
421         * @throws IllegalArgumentException
422         *             if <code>position</code> is less than 0.
423         * @throws IOException
424         *             if another I/O error occurs.
425         * @throws NonReadableChannelException
426         *             if the channel has not been opened in a mode that permits
427         *             reading.
428         * @since Android 1.0
429         */
430        public abstract int read(ByteBuffer buffer, long position)
431                throws IOException;

433        /**
434         * Reads bytes from this file channel and stores them in the specified array
435         * of buffers. This method attempts to read as many bytes as can be stored
436         * in the buffer array from this channel and returns the number of bytes
437         * actually read. It also increases the file position by the number of bytes
438         * read.
439         * <p>
440         * If a read operation is in progress, subsequent threads will block until
441         * the read is completed and will then contend for the ability to read.
442         * </p>
443         * <p>
444         * Calling this method is equivalent to calling
445         * {@code read(buffers, 0, buffers.length);}
446         * </p>
447         * 
448         * @param buffers
449         *            the array of byte buffers into which the bytes will be copied.
450         * @return the number of bytes actually read.
451         * @throws AsynchronousCloseException
452         *             if this channel is closed by another thread during this read
453         *             operation.
454         * @throws ClosedByInterruptException
455         *             if the thread is interrupted by another thread during this
456         *             read operation.
457         * @throws ClosedChannelException
458         *             if this channel is closed.
459         * @throws IOException
460         *             if another I/O error occurs; details are in the message.
461         * @throws NonReadableChannelException
462         *             if the channel has not been opened in a mode that permits
463         *             reading.
464         * @since Android 1.0
465         */
466        public final long read(ByteBuffer[] buffers) throws IOException {
467            return read(buffers, 0, buffers.length);
468        }

544         * @throws NonReadableChannelException
545         *             if the source channel is not readable.
546         * @throws NonWritableChannelException
547         *             if this channel is not writable.
548         * @throws ClosedChannelException
549         *             if either channel has already been closed.
550         * @throws AsynchronousCloseException
551         *             if either channel is closed by other threads during this
552         *             operation.
553         * @throws ClosedByInterruptException
554         *             if the thread is interrupted during this operation.
555         * @throws IOException
556         *             if any I/O error occurs.
557         * @since Android 1.0

642         *             if a lock already exists that overlaps this lock request or

653        /**
654         * Attempts to acquire an exclusive lock on this file without blocking. The
655         * method returns {@code null} if the acquisition would result in an
656         * overlapped lock with another OS process.
657         * <p>
658         * It is possible to acquire a lock for any region even if it's completely
659         * outside of the file's size. The size of the lock is fixed. If the file
660         * grows outside of the lock that region of the file won't be locked by this
661         * lock.
662         * </p>
663         * 
664         * @param position
665         *            the starting position.
666         * @param size
667         *            the size of file to lock.
668         * @param shared
669         *            true if the lock is shared.
670         * @return the file lock object, or {@code null} if the lock would overlap
671         *         with an existing exclusive lock in another OS process.
672         * @throws IllegalArgumentException
673         *             if any parameters are invalid.
674         * @throws ClosedChannelException
675         *             if the file channel is closed.
676         * @throws OverlappingFileLockException
677         *             if a lock is already held that overlaps this lock request or
678         *             another thread is waiting to acquire a lock that will overlap
679         *             with this request.
680         * @throws IOException
681         *             if any I/O error occurs.
682         * @since Android 1.0
683         */
684        public abstract FileLock tryLock(long position, long size, boolean shared)
685                throws IOException;

714        /**
715         * Writes bytes from the given buffer to this file channel starting at the
716         * given file position.
717         * <p>
718         * The bytes are written starting at the given file position (up to the
719         * remaining number of bytes in the buffer). The number of bytes actually
720         * written is returned.
721         * </p>
722         * <p>

724         * extended up to the given position by the required number of unspecified

727         * <p>
728         * Note that the file position is not modified by this method.
729         * </p>
730         * 
731         * @param buffer

733         * @param position

735         * @return the number of bytes actually written.
736         * @throws IllegalArgumentException
737         *             if <code>position</code> is less than 0.
738         * @throws ClosedChannelException
739         *             if this channel is closed.

742         * @throws AsynchronousCloseException
743         *             if this channel is closed by another thread while this method
744         *             is executing.
745         * @throws ClosedByInterruptException
746         *             if another thread interrupts the calling thread while this
747         *             operation is in progress. The interrupt state of the calling
748         *             thread is set and the channel is closed.
749         * @throws IOException

751         * @since Android 1.0
752         */
753        public abstract int write(ByteBuffer buffer, long position)
754                throws IOException;

757         * Writes bytes from all the given byte buffers to this file channel.
758         * <p>
759         * The bytes are written starting at the current file position, and after
760         * the bytes are written (up to the remaining number of bytes in all the
761         * buffers), the file position is increased by the number of bytes actually
762         * written.
763         * <p>
