047    /**
048     * A concrete subclass of {@link NumberFormat} that formats decimal numbers. It
049     * has a variety of features designed to make it possible to parse and format
050     * numbers in any locale, including support for Western, Arabic, or Indic
051     * digits. It also supports different flavors of numbers, including integers
052     * ("123"), fixed-point numbers ("123.4"), scientific notation ("1.23E4"),
053     * percentages ("12%"), and currency amounts ("$123"). All of these flavors can
054     * be easily localized.
055     * <p>
056     * <strong>This is an enhanced version of {@code DecimalFormat} that is based on
057     * the standard version in the RI. New or changed functionality is labeled
058     * <strong><font color="red">NEW</font></strong>.</strong>
059     * </p>
060     * <p>
061     * To obtain a {@link NumberFormat} for a specific locale (including the default
062     * locale), call one of {@code NumberFormat}'s factory methods such as
063     * {@code NumberFormat.getInstance}. Do not call the {@code DecimalFormat}
064     * constructors directly, unless you know what you are doing, since the
065     * {@link NumberFormat} factory methods may return subclasses other than
066     * {@code DecimalFormat}. If you need to customize the format object, do
067     * something like this: <blockquote>
068     * 
069     * <pre>
070     * NumberFormat f = NumberFormat.getInstance(loc);
071     * if (f instanceof DecimalFormat) {
072     *     ((DecimalFormat)f).setDecimalSeparatorAlwaysShown(true);
073     * }
074     * </pre>
075     * 
076     * </blockquote>
077     * <h5>Example:</h5>
078     * <blockquote>
079     * 
080     * <pre>
081     * // Print out a number using the localized number, currency,
082     * // and percent format for each locale
083     * Locale[] locales = NumberFormat.getAvailableLocales();
084     * double myNumber = -1234.56;
085     * NumberFormat format;
086     * for (int j = 0; j &lt; 3; ++j) {
087     *     System.out.println(&quot;FORMAT&quot;);
088     *     for (int i = 0; i &lt; locales.length; ++i) {
089     *         if (locales[i].getCountry().length() == 0) {
090     *             // Skip language-only locales
091     *             continue;
092     *         }
093     *         System.out.print(locales[i].getDisplayName());
094     *         switch (j) {
095     *             case 0:
096     *                 format = NumberFormat.getInstance(locales[i]);
097     *                 break;
098     *             case 1:
099     *                 format = NumberFormat.getCurrencyInstance(locales[i]);
100     *                 break;
101     *             default:
102     *                 format = NumberFormat.getPercentInstance(locales[i]);
103     *                 break;
104     *         }
105     *         try {
106     *             // Assume format is a DecimalFormat
107     *             System.out.print(&quot;: &quot;; + ((DecimalFormat)format).toPattern() + &quot; -&gt; &quot;
108     *                     + form.format(myNumber));
109     *         } catch (Exception e) {
110     *         }
111     *         try {
112     *             System.out.println(&quot; -&gt; &quot; + format.parse(form.format(myNumber)));
113     *         } catch (ParseException e) {
114     *         }
115     *     }
116     * }
117     * </pre>
118     * 
119     * </blockquote>
120     * <h4>Patterns</h4>
121     * <p>
122     * A {@code DecimalFormat} consists of a <em>pattern</em> and a set of
123     * <em>symbols</em>. The pattern may be set directly using
124     * {@link #applyPattern(String)}, or indirectly using other API methods which
125     * manipulate aspects of the pattern, such as the minimum number of integer
126     * digits. The symbols are stored in a {@link DecimalFormatSymbols} object. When
127     * using the {@link NumberFormat} factory methods, the pattern and symbols are
128     * read from ICU's locale data.
129     * </p>
130     * <h4>Special Pattern Characters</h4>
131     * <p>
132     * Many characters in a pattern are taken literally; they are matched during
133     * parsing and are written out unchanged during formatting. On the other hand,
134     * special characters stand for other characters, strings, or classes of
135     * characters. For example, the '#' character is replaced by a localized digit.
136     * Often the replacement character is the same as the pattern character; in the
137     * U.S. locale, the ',' grouping character is replaced by ','. However, the
138     * replacement is still happening, and if the symbols are modified, the grouping
139     * character changes. Some special characters affect the behavior of the
140     * formatter by their presence; for example, if the percent character is seen,
141     * then the value is multiplied by 100 before being displayed.
142     * </p>
143     * <p>
144     * To insert a special character in a pattern as a literal, that is, without any
145     * special meaning, the character must be quoted. There are some exceptions to
146     * this which are noted below.
147     * </p>
148     * <p>
149     * The characters listed here are used in non-localized patterns. Localized
150     * patterns use the corresponding characters taken from this formatter's
151     * {@link DecimalFormatSymbols} object instead, and these characters lose their
152     * special status. Two exceptions are the currency sign and quote, which are not
153     * localized.

257     * <p>
258     * A {@code DecimalFormat} pattern contains a postive and negative subpattern,
259     * for example, "#,##0.00;(#,##0.00)". Each subpattern has a prefix, a numeric
260     * part and a suffix. If there is no explicit negative subpattern, the negative
261     * subpattern is the localized minus sign prefixed to the positive subpattern.
262     * That is, "0.00" alone is equivalent to "0.00;-0.00". If there is an explicit
263     * negative subpattern, it serves only to specify the negative prefix and
264     * suffix; the number of digits, minimal digits, and other characteristics are
265     * ignored in the negative subpattern. This means that "#,##0.0#;(#)" produces
266     * precisely the same result as "#,##0.0#;(#,##0.0#)".
267     * <p>
268     * The prefixes, suffixes, and various symbols used for infinity, digits,
269     * thousands separators, decimal separators, etc. may be set to arbitrary
270     * values, and they will appear properly during formatting. However, care must
271     * be taken that the symbols and strings do not conflict, or parsing will be
272     * unreliable. For example, either the positive and negative prefixes or the
273     * suffixes must be distinct for {@link #parse} to be able to distinguish
274     * positive from negative values. Another example is that the decimal separator
275     * and thousands separator should be distinct characters, or parsing will be
276     * impossible.
277     * <p>
278     * The <em>grouping separator</em> is a character that separates clusters of
279     * integer digits to make large numbers more legible. It is commonly used for
280     * thousands, but in some locales it separates ten-thousands. The <em>grouping
281     * size</em>
282     * is the number of digits between the grouping separators, such as 3 for
283     * "100,000,000" or 4 for "1 0000 0000". There are actually two different
284     * grouping sizes: One used for the least significant integer digits, the
285     * <em>primary grouping size</em>, and one used for all others, the
286     * <em>secondary grouping size</em>. In most locales these are the same, but
287     * sometimes they are different. For example, if the primary grouping interval
288     * is 3, and the secondary is 2, then this corresponds to the pattern
289     * "#,##,##0", and the number 123456789 is formatted as "12,34,56,789". If a
290     * pattern contains multiple grouping separators, the interval between the last
291     * one and the end of the integer defines the primary grouping size, and the
292     * interval between the last two defines the secondary grouping size. All others
293     * are ignored, so "#,##,###,####", "###,###,####" and "##,#,###,####" produce
294     * the same result.
295     * <p>
296     * Illegal patterns, such as "#.#.#" or "#.###,###", will cause
297     * {@code DecimalFormat} to throw an {@link IllegalArgumentException} with a
298     * message that describes the problem.
299     * <h4>Pattern BNF</h4>
300     * 
301     * <pre>
302     * pattern    := subpattern (';' subpattern)?
303     * subpattern := prefix? number exponent? suffix?
304     * number     := (integer ('.' fraction)?) | sigDigits
305     * prefix     := '\\u0000'..'\\uFFFD' - specialCharacters
306     * suffix     := '\\u0000'..'\\uFFFD' - specialCharacters
307     * integer    := '#'* '0'* '0'
308     * fraction   := '0'* '#'*
309     * sigDigits  := '#'* '@' '@'* '#'*
310     * exponent   := 'E' '+'? '0'* '0'
311     * padSpec    := '*' padChar
312     * padChar    := '\\u0000'..'\\uFFFD' - quote
313     *  
314     * Notation:
315     *   X*       0 or more instances of X
316     *   X?       0 or 1 instances of X
317     *   X|Y      either X or Y
318     *   C..D     any character from C up to D, inclusive
319     *   S-T      characters in S, except those in T
320     * </pre>
321     * 
322     * The first subpattern is for positive numbers. The second (optional)
323     * subpattern is for negative numbers.
324     * <p>
325     * Not indicated in the BNF syntax above:
326     * <ul>
327     * <li>The grouping separator ',' can occur inside the integer and sigDigits
328     * elements, between any two pattern characters of that element, as long as the
329     * integer or sigDigits element is not followed by the exponent element.
330     * <li><font color="red"><strong>NEW</strong>&nbsp;</font> Two
331     * grouping intervals are recognized: The one between the decimal point and the
332     * first grouping symbol and the one between the first and second grouping
333     * symbols. These intervals are identical in most locales, but in some locales
334     * they differ. For example, the pattern &quot;#,##,###&quot; formats the number
335     * 123456789 as &quot;12,34,56,789&quot;.</li>
336     * <li> <strong><font color="red">NEW</font>&nbsp;</strong> The pad
337     * specifier {@code padSpec} may appear before the prefix, after the prefix,
338     * before the suffix, after the suffix or not at all.
339     * </ul>
340     * <h4>Parsing</h4>
341     * <p>
342     * {@code DecimalFormat} parses all Unicode characters that represent decimal
343     * digits, as defined by {@link Character#digit(int, int)}. In addition,
344     * {@code DecimalFormat} also recognizes as digits the ten consecutive
345     * characters starting with the localized zero digit defined in the
346     * {@link DecimalFormatSymbols} object. During formatting, the
347     * {@link DecimalFormatSymbols}-based digits are written out.
348     * <p>
349     * During parsing, grouping separators are ignored.
350     * <p>
351     * If {@link #parse(String, ParsePosition)} fails to parse a string, it returns
352     * {@code null} and leaves the parse position unchanged.
353     * <h4>Formatting</h4>
354     * <p>
355     * Formatting is guided by several parameters, all of which can be specified
356     * either using a pattern or using the API. The following description applies to
357     * formats that do not use <a href="#sci">scientific notation</a> or <a
358     * href="#sigdig">significant digits</a>.
359     * <ul>
360     * <li>If the number of actual integer digits exceeds the
361     * <em>maximum integer digits</em>, then only the least significant digits
362     * are shown. For example, 1997 is formatted as "97" if maximum integer digits
363     * is set to 2.
364     * <li>If the number of actual integer digits is less than the
365     * <em>minimum integer digits</em>, then leading zeros are added. For
366     * example, 1997 is formatted as "01997" if minimum integer digits is set to 5.
367     * <li>If the number of actual fraction digits exceeds the <em>maximum
368     * fraction digits</em>,
369     * then half-even rounding is performed to the maximum fraction digits. For
370     * example, 0.125 is formatted as "0.12" if the maximum fraction digits is 2.
371     * <li>If the number of actual fraction digits is less than the
372     * <em>minimum fraction digits</em>, then trailing zeros are added. For
373     * example, 0.125 is formatted as "0.1250" if the mimimum fraction digits is set
374     * to 4.
375     * <li>Trailing fractional zeros are not displayed if they occur <em>j</em>
376     * positions after the decimal, where <em>j</em> is less than the maximum
377     * fraction digits. For example, 0.10004 is formatted as "0.1" if the maximum
378     * fraction digits is four or less.
379     * </ul>
380     * <p>
381     * <strong>Special Values</strong>
382     * <p>
383     * {@code NaN} is represented as a single character, typically
384     * {@code &#92;uFFFD}. This character is determined by the
385     * {@link DecimalFormatSymbols} object. This is the only value for which the
386     * prefixes and suffixes are not used.
387     * <p>
388     * Infinity is represented as a single character, typically {@code &#92;u221E},
389     * with the positive or negative prefixes and suffixes applied. The infinity
390     * character is determined by the {@link DecimalFormatSymbols} object. <a
391     * name="sci">
392     * <h4>Scientific Notation</h4>
393     * </a>
394     * <p>
395     * Numbers in scientific notation are expressed as the product of a mantissa and
396     * a power of ten, for example, 1234 can be expressed as 1.234 x 10<sup>3</sup>.
397     * The mantissa is typically in the half-open interval [1.0, 10.0) or sometimes
398     * [0.0, 1.0), but it does not need to be. {@code DecimalFormat} supports
399     * arbitrary mantissas. {@code DecimalFormat} can be instructed to use
400     * scientific notation through the API or through the pattern. In a pattern, the
401     * exponent character immediately followed by one or more digit characters
402     * indicates scientific notation. Example: "0.###E0" formats the number 1234 as
403     * "1.234E3".
404     * <ul>
405     * <li>The number of digit characters after the exponent character gives the
406     * minimum exponent digit count. There is no maximum. Negative exponents are
407     * formatted using the localized minus sign, <em>not</em> the prefix and
408     * suffix from the pattern. This allows patterns such as "0.###E0 m/s". To
409     * prefix positive exponents with a localized plus sign, specify '+' between the
410     * exponent and the digits: "0.###E+0" will produce formats "1E+1", "1E+0",
411     * "1E-1", etc. (In localized patterns, use the localized plus sign rather than
412     * '+'.)
413     * <li>The minimum number of integer digits is achieved by adjusting the
414     * exponent. Example: 0.00123 formatted with "00.###E0" yields "12.3E-4". This
415     * only happens if there is no maximum number of integer digits. If there is a
416     * maximum, then the minimum number of integer digits is fixed at one.
417     * <li>The maximum number of integer digits, if present, specifies the exponent
418     * grouping. The most common use of this is to generate <em>engineering
419     * notation</em>,
420     * in which the exponent is a multiple of three, e.g., "##0.###E0". The number
421     * 12345 is formatted using "##0.###E0" as "12.345E3".
422     * <li>When using scientific notation, the formatter controls the digit counts
423     * using significant digits logic. The maximum number of significant digits
424     * limits the total number of integer and fraction digits that will be shown in
425     * the mantissa; it does not affect parsing. For example, 12345 formatted with
426     * "##0.##E0" is "12.3E3". See the section on significant digits for more
427     * details.
428     * <li>The number of significant digits shown is determined as follows: If no
429     * significant digits are used in the pattern then the minimum number of
430     * significant digits shown is one, the maximum number of significant digits
431     * shown is the sum of the <em>minimum integer</em> and
432     * <em>maximum fraction</em> digits, and it is unaffected by the maximum
433     * integer digits. If this sum is zero, then all significant digits are shown.
434     * If significant digits are used in the pattern then the number of integer
435     * digits is fixed at one and there is no exponent grouping.
436     * <li>Exponential patterns may not contain grouping separators.
437     * </ul>
438     * <a name="sigdig">
439     * <h4> <strong><font color="red">NEW</font>&nbsp;</strong> Significant
440     * Digits</h4>
441     * <p>
442     * </a> {@code DecimalFormat} has two ways of controlling how many digits are
443     * shown: (a) significant digit counts or (b) integer and fraction digit counts.
444     * Integer and fraction digit counts are described above. When a formatter uses
445     * significant digits counts, the number of integer and fraction digits is not
446     * specified directly, and the formatter settings for these counts are ignored.
447     * Instead, the formatter uses as many integer and fraction digits as required
448     * to display the specified number of significant digits.
449     * </p>
450     * <h5>Examples:</h5>
451     * <blockquote> <table border=0 cellspacing=3 cellpadding=0>
452     * <tr bgcolor="#ccccff">
453     * <th align="left">Pattern</th>
454     * <th align="left">Minimum significant digits</th>
455     * <th align="left">Maximum significant digits</th>
456     * <th align="left">Number</th>
457     * <th align="left">Output of format()</th>
458     * </tr>
459     * <tr valign="top">
460     * <td>{@code @@@}
461     * <td>3</td>
462     * <td>3</td>
463     * <td>12345</td>
464     * <td>{@code 12300}</td>
465     * </tr>
466     * <tr valign="top" bgcolor="#eeeeff">
467     * <td>{@code @@@}</td>
468     * <td>3</td>
469     * <td>3</td>
470     * <td>0.12345</td>
471     * <td>{@code 0.123}</td>
472     * </tr>
473     * <tr valign="top">
474     * <td>{@code @@##}</td>
475     * <td>2</td>
476     * <td>4</td>
477     * <td>3.14159</td>
478     * <td>{@code 3.142}</td>
479     * </tr>
480     * <tr valign="top" bgcolor="#eeeeff">
481     * <td>{@code @@##}</td>
482     * <td>2</td>
483     * <td>4</td>
484     * <td>1.23004</td>
485     * <td>{@code 1.23}</td>
486     * </tr>
487     * </table> </blockquote>
488     * <ul>
489     * <li>Significant digit counts may be expressed using patterns that specify a
490     * minimum and maximum number of significant digits. These are indicated by the
491     * {@code '@'} and {@code '#'} characters. The minimum number of significant
492     * digits is the number of {@code '@'} characters. The maximum number of
493     * significant digits is the number of {@code '@'} characters plus the number of
494     * {@code '#'} characters following on the right. For example, the pattern
495     * {@code "@@@"} indicates exactly 3 significant digits. The pattern
496     * {@code "@##"} indicates from 1 to 3 significant digits. Trailing zero digits
497     * to the right of the decimal separator are suppressed after the minimum number
498     * of significant digits have been shown. For example, the pattern {@code "@##"}
499     * formats the number 0.1203 as {@code "0.12"}.
500     * <li>If a pattern uses significant digits, it may not contain a decimal
501     * separator, nor the {@code '0'} pattern character. Patterns such as
502     * {@code "@00"} or {@code "@.###"} are disallowed.
503     * <li>Any number of {@code '#'} characters may be prepended to the left of the
504     * leftmost {@code '@'} character. These have no effect on the minimum and
505     * maximum significant digit counts, but may be used to position grouping
506     * separators. For example, {@code "#,#@#"} indicates a minimum of one
507     * significant digit, a maximum of two significant digits, and a grouping size
508     * of three.
509     * <li>In order to enable significant digits formatting, use a pattern
510     * containing the {@code '@'} pattern character.
511     * <li>In order to disable significant digits formatting, use a pattern that
512     * does not contain the {@code '@'} pattern character.
513     * <li>The number of significant digits has no effect on parsing.
514     * <li>Significant digits may be used together with exponential notation. Such
515     * patterns are equivalent to a normal exponential pattern with a minimum and
516     * maximum integer digit count of one, a minimum fraction digit count of the
517     * number of '@' characters in the pattern - 1, and a maximum fraction digit
518     * count of the number of '@' and '#' characters in the pattern - 1. For
519     * example, the pattern {@code "@@###E0"} is equivalent to {@code "0.0###E0"}.
520     * <li>If signficant digits are in use then the integer and fraction digit
521     * counts, as set via the API, are ignored.
522     * </ul>
523     * <h4> <strong><font color="red">NEW</font>&nbsp;</strong> Padding</h4>
524     * <p>
525     * {@code DecimalFormat} supports padding the result of {@code format} to a
526     * specific width. Padding may be specified either through the API or through
527     * the pattern syntax. In a pattern, the pad escape character followed by a
528     * single pad character causes padding to be parsed and formatted. The pad
529     * escape character is '*' in unlocalized patterns. For example,
530     * {@code "$*x#,##0.00"} formats 123 to {@code "$xx123.00"}, and 1234 to
531     * {@code "$1,234.00"}.
532     * <ul>
533     * <li>When padding is in effect, the width of the positive subpattern,
534     * including prefix and suffix, determines the format width. For example, in the
535     * pattern {@code "* #0 o''clock"}, the format width is 10.</li>
536     * <li>The width is counted in 16-bit code units (Java {@code char}s).</li>
537     * <li>Some parameters which usually do not matter have meaning when padding is
538     * used, because the pattern width is significant with padding. In the pattern "*
539     * ##,##,#,##0.##", the format width is 14. The initial characters "##,##," do
540     * not affect the grouping size or maximum integer digits, but they do affect
541     * the format width.</li>
542     * <li>Padding may be inserted at one of four locations: before the prefix,
543     * after the prefix, before the suffix or after the suffix. If padding is
544     * specified in any other location, {@link #applyPattern} throws an {@link
545     * IllegalArgumentException}. If there is no prefix, before the prefix and after
546     * the prefix are equivalent, likewise for the suffix.</li>
547     * <li>When specified in a pattern, the 16-bit {@code char} immediately
548     * following the pad escape is the pad character. This may be any character,
549     * including a special pattern character. That is, the pad escape
550     * <em>escapes</em> the following character. If there is no character after
551     * the pad escape, then the pattern is illegal.</li>
552     * </ul>
553     * <h4>Synchronization</h4>

556     * not access one formatter concurrently.
