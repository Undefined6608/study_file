043    /**
044     * The abstract base class for all number formats. This class provides the
045     * interface for formatting and parsing numbers. {@code NumberFormat} also
046     * provides methods for determining which locales have number formats, and what
047     * their names are.
048     * <p>
049     * {@code NumberFormat} helps you to format and parse numbers for any locale.
050     * Your code can be completely independent of the locale conventions for decimal
051     * points, thousands-separators, or even the particular decimal digits used, or
052     * whether the number format is even decimal.
053     * </p>
054     * <p>
055     * To format a number for the current locale, use one of the factory class
056     * methods:
057     * </p>
058     * <blockquote>
059     * 
060     * <pre>
061     * myString = NumberFormat.getInstance().format(myNumber);
062     * </pre>
063     * 
064     * </blockquote>
065     * <p>
066     * If you are formatting multiple numbers, it is more efficient to get the
067     * format and use it multiple times so that the system doesn't have to fetch the
068     * information about the local language and country conventions multiple times.
069     * </p>
070     * <blockquote>
071     * 
072     * <pre>
073     * NumberFormat nf = NumberFormat.getInstance();
074     * for (int i = 0; i &lt; a.length; ++i) {
075     *     output.println(nf.format(myNumber[i]) + &quot;; &quot;);
076     * }
077     * </pre>
078     * 
079     * </blockquote>
080     * <p>
081     * To format a number for a different locale, specify it in the call to
082     * {@code getInstance}.
083     * </p>
084     * <blockquote>
085     * 
086     * <pre>
087     * NumberFormat nf = NumberFormat.getInstance(Locale.FRENCH);
088     * </pre>
089     * 
090     * </blockquote>
091     * <p>
092     * You can also use a {@code NumberFormat} to parse numbers:
093     * </p>
094     * <blockquote>
095     * 
096     * <pre>
097     * myNumber = nf.parse(myString);
098     * </pre>
099     * 
100     * </blockquote>
101     * <p>
102     * Use {@code getInstance} or {@code getNumberInstance} to get the normal number
103     * format. Use {@code getIntegerInstance} to get an integer number format. Use
104     * {@code getCurrencyInstance} to get the currency number format and use
105     * {@code getPercentInstance} to get a format for displaying percentages. With
106     * this format, a fraction like 0.53 is displayed as 53%.
107     * </p>
108     * <p>
109     * You can also control the display of numbers with methods such as
110     * {@code setMinimumFractionDigits}. If you want even more control over the
111     * format or parsing, or want to give your users more control, you can try
112     * casting the {@code NumberFormat} you get from the factory methods to a
113     * {@code DecimalFormat}. This will work for the vast majority of locales; just
114     * remember to put it in a {@code try} block in case you encounter an unusual
115     * one.
116     * </p>
117     * <p>
118     * {@code NumberFormat} is designed such that some controls work for formatting
119     * and others work for parsing. For example, {@code setParseIntegerOnly} only
120     * affects parsing: If set to {@code true}, "3456.78" is parsed as 3456 (and
121     * leaves the parse position just after '6'); if set to {@code false},
122     * "3456.78" is parsed as 3456.78 (and leaves the parse position just after
123     * '8'). This is independent of formatting.
124     * </p>
125     * <p>
126     * You can also use forms of the {@code parse} and {@code format} methods with
127     * {@code ParsePosition} and {@code FieldPosition} to allow you to:
128     * <ul>
129     * <li>progressively parse through pieces of a string;</li>
130     * <li>align the decimal point and other areas.</li>
131     * </ul>
132     * For example, you can align numbers in two ways:
133     * <ol>
134     * <li> If you are using a monospaced font with spacing for alignment, you can
135     * pass the {@code FieldPosition} in your format call, with {@code field} =
136     * {@code INTEGER_FIELD}. On output, {@code getEndIndex} will be set to the
137     * offset between the last character of the integer and the decimal. Add
138     * (desiredSpaceCount - getEndIndex) spaces to the front of the string.</li>
139     * <li> If you are using proportional fonts, instead of padding with spaces,
140     * measure the width of the string in pixels from the start to
141     * {@code getEndIndex}. Then move the pen by (desiredPixelWidth -
142     * widthToAlignmentPoint) before drawing the text. This also works where there
143     * is no decimal but possibly additional characters before or after the number,
144     * for example with parentheses in negative numbers: "(12)" for -12.</li>
145     * </ol>
146     * <h4>Synchronization</h4>
147     * <p>
148     * Number formats are generally not synchronized. It is recommended to create
149     * separate format instances for each thread. If multiple threads access a
150     * format concurrently, it must be synchronized externally.
151     * <p>
152     * <h4>DecimalFormat</h4>
153     * <p>
154     * {@code DecimalFormat} is the concrete implementation of {@code NumberFormat},
155     * and the {@code NumberFormat} API is essentially an abstraction of
156     * {@code DecimalFormat's} API. Refer to {@code DecimalFormat} for more
157     * information about this API.
158     * </p>
159     * 
160     * @see DecimalFormat
161     * @see java.text.ChoiceFormat
162     * @since Android 1.0
163     */

311        /**
312         * Formats the specified object as a string using the pattern of this number
313         * format and appends the string to the specified string buffer.
314         * <p>
315         * If the {@code field} member of {@code field} contains a value specifying
316         * a format field, then its {@code beginIndex} and {@code endIndex} members
317         * will be updated with the position of the first occurrence of this field
318         * in the formatted text.
319         * </p>
320         * 
321         * @param object
322         *            the object to format, must be a {@code Number}.
323         * @param buffer
324         *            the target string buffer to append the formatted number to.
325         * @param field
326         *            on input: an optional alignment field; on output: the offsets
327         *            of the alignment field in the formatted text.
328         * @return the string buffer.
329         * @throws IllegalArgumentException
330         *             if {@code object} is not an instance of {@code Number}.
331         * @since Android 1.0
332         */
333        @Override
334        public StringBuffer format(Object object, StringBuffer buffer,
335                FieldPosition field) {
336            if (object instanceof Number) {
337                double dv = ((Number) object).doubleValue();
338                long lv = ((Number) object).longValue();
339                if (dv == lv) {
340                    return format(lv, buffer, field);
341                }
342                return format(dv, buffer, field);
343            }
344            throw new IllegalArgumentException();
345        }

583        /**
584         * Parses a {@code Number} from the specified string using the rules of this
585         * number format.
586         * 
587         * @param string
588         *            the string to parse.
589         * @return the {@code Number} resulting from the parsing.
590         * @exception ParseException
591         *                if an error occurs during parsing.
592         * @since Android 1.0
593         */
594        public Number parse(String string) throws ParseException {
595            ParsePosition pos = new ParsePosition(0);
596            Number number = parse(string, pos);
597            if (pos.getErrorIndex() != -1 || pos.getIndex() == 0) {
598                throw new ParseException(null, pos.getErrorIndex());
599            }
600            return number;
601        }

604         * Parses a {@code Number} from the specified string starting at the index
605         * specified by {@code position}. If the string is successfully parsed then
606         * the index of the {@code ParsePosition} is updated to the index following
607         * the parsed text. On error, the index is unchanged and the error index of
608         * {@code ParsePosition} is set to the index where the error occurred.
609         * 
610         * @param string
611         *            the string to parse.
612         * @param position
613         *            input/output parameter, specifies the start index in
614         *            {@code string} from where to start parsing. If parsing is
615         *            successful, it is updated with the index following the parsed
616         *            text; on error, the index is unchanged and the error index is
617         *            set to the index where the error occurred.
618         * @return the {@code Number} resulting from the parse or {@code null} if
619         *         there is an error.
620         * @since Android 1.0
621         */
622        public abstract Number parse(String string, ParsePosition position);

624        @Override
625        public final Object parseObject(String string, ParsePosition position) {
626            if (position == null) {
627                // text.1A=position is null
628                throw new NullPointerException(Messages.getString("text.1A")); //$NON-NLS-1$
629            }
630    
631            try {
632                return parse(string, position);
633            } catch (Exception e) {
634                return null;
635            }
636        }

791        private void readObject(ObjectInputStream stream) throws IOException,
792                ClassNotFoundException {
793            ObjectInputStream.GetField fields = stream.readFields();
794            groupingUsed = fields.get("groupingUsed", true); //$NON-NLS-1$
795            parseIntegerOnly = fields.get("parseIntegerOnly", false); //$NON-NLS-1$
796            if (fields.get("serialVersionOnStream", 0) == 0) { //$NON-NLS-1$
797                maximumFractionDigits = fields.get("maxFractionDigits", (byte) 3); //$NON-NLS-1$
798                maximumIntegerDigits = fields.get("maxIntegerDigits", (byte) 40); //$NON-NLS-1$
799                minimumFractionDigits = fields.get("minFractionDigits", (byte) 0); //$NON-NLS-1$
800                minimumIntegerDigits = fields.get("minIntegerDigits", (byte) 1); //$NON-NLS-1$
801            } else {
802                maximumFractionDigits = fields.get("maximumFractionDigits", 3); //$NON-NLS-1$
803                maximumIntegerDigits = fields.get("maximumIntegerDigits", 40); //$NON-NLS-1$
804                minimumFractionDigits = fields.get("minimumFractionDigits", 0); //$NON-NLS-1$
805                minimumIntegerDigits = fields.get("minimumIntegerDigits", 1); //$NON-NLS-1$
806            }
807            if (minimumIntegerDigits > maximumIntegerDigits
808                    || minimumFractionDigits > maximumFractionDigits) {
809                // text.00=min digits greater than max digits
810                throw new InvalidObjectException(Messages.getString("text.00")); //$NON-NLS-1$
811            }
812            if (minimumIntegerDigits < 0 || maximumIntegerDigits < 0
813                    || minimumFractionDigits < 0 || maximumFractionDigits < 0) {
814                // text.01=min or max digits negative
815                throw new InvalidObjectException(Messages.getString("text.01")); //$NON-NLS-1$
816            }
817        }
