020    /**
021     * This interface represents an entity, either parsed or unparsed, in an XML 
022     * document. Note that this models the entity itself not the entity 
023     * declaration. <code>Entity</code> declaration modeling has been left for a 
024     * later Level of the DOM specification.
025     * <p>The <code>nodeName</code> attribute that is inherited from 
026     * <code>Node</code> contains the name of the entity.
027     * <p>An XML processor may choose to completely expand entities before the 
028     * structure model is passed to the DOM; in this case there will be no 
029     * <code>EntityReference</code> nodes in the document tree.
030     * <p>XML does not mandate that a non-validating XML processor read and 
031     * process entity declarations made in the external subset or declared in 
032     * external parameter entities. This means that parsed entities declared in 
033     * the external subset need not be expanded by some classes of applications, 
034     * and that the replacement value of the entity may not be available. When 
035     * the replacement value is available, the corresponding <code>Entity</code> 
036     * node's child list represents the structure of that replacement text. 
037     * Otherwise, the child list is empty.
038     * <p>The DOM Level 2 does not support editing <code>Entity</code> nodes; if a 
039     * user wants to make changes to the contents of an <code>Entity</code>, 
040     * every related <code>EntityReference</code> node has to be replaced in the 
041     * structure model by a clone of the <code>Entity</code>'s contents, and 
042     * then the desired changes must be made to each of those clones instead. 
043     * <code>Entity</code> nodes and all their descendants are readonly.
044     * <p>An <code>Entity</code> node does not have any parent.If the entity 
045     * contains an unbound namespace prefix, the <code>namespaceURI</code> of 
046     * the corresponding node in the <code>Entity</code> node subtree is 
047     * <code>null</code>. The same is true for <code>EntityReference</code> 
048     * nodes that refer to this entity, when they are created using the 
049     * <code>createEntityReference</code> method of the <code>Document</code> 
050     * interface. The DOM Level 2 does not support any mechanism to resolve 
051     * namespace prefixes.
052     * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>.
053     */
054    public interface Entity extends Node {
055        /**
056         * The public identifier associated with the entity, if specified. If the 
057         * public identifier was not specified, this is <code>null</code>.
058         * 
059         * @return the public identifier of the entity or <code>null</code>, if the
060         * entity does not have a public identifier.
061         */
062        public String getPublicId();
063    
064        /**
065         * The system identifier associated with the entity, if specified. If the 
066         * system identifier was not specified, this is <code>null</code>.
067         * 
068         * @return the system identifier of the entity or <code>null</code>, if the
069         * entity does not have a system identifier.
070         */
071        public String getSystemId();
072    
073        /**
074         * For unparsed entities, the name of the notation for the entity. For 
075         * parsed entities, this is <code>null</code>. 
076         * 
077         * @return the notation name of the entity or <code>null</code>, if the
078         * entity is parsed.
079         */
080        public String getNotationName();
081    
082    }
