020    /**
021     *  The <code>Attr</code> interface represents an attribute in an 
022     * <code>Element</code> object. Typically the allowable values for the 
023     * attribute are defined in a document type definition.
024     * <p><code>Attr</code> objects inherit the <code>Node</code> interface, but 
025     * since they are not actually child nodes of the element they describe, the 
026     * DOM does not consider them part of the document tree. Thus, the 
027     * <code>Node</code> attributes <code>parentNode</code>, 
028     * <code>previousSibling</code>, and <code>nextSibling</code> have a 
029     * <code>null</code> value for <code>Attr</code> objects. The DOM takes the 
030     * view that attributes are properties of elements rather than having a 
031     * separate identity from the elements they are associated with; this should 
032     * make it more efficient to implement such features as default attributes 
033     * associated with all elements of a given type. Furthermore, 
034     * <code>Attr</code> nodes may not be immediate children of a 
035     * <code>DocumentFragment</code>. However, they can be associated with 
036     * <code>Element</code> nodes contained within a 
037     * <code>DocumentFragment</code>. In short, users and implementors of the 
038     * DOM need to be aware that <code>Attr</code> nodes have some things in 
039     * common with other objects inheriting the <code>Node</code> interface, but 
040     * they also are quite distinct.
041     * <p> The attribute's effective value is determined as follows: if this 
042     * attribute has been explicitly assigned any value, that value is the 
043     * attribute's effective value; otherwise, if there is a declaration for 
044     * this attribute, and that declaration includes a default value, then that 
045     * default value is the attribute's effective value; otherwise, the 
046     * attribute does not exist on this element in the structure model until it 
047     * has been explicitly added. Note that the <code>nodeValue</code> attribute 
048     * on the <code>Attr</code> instance can also be used to retrieve the string 
049     * version of the attribute's value(s). 
050     * <p>In XML, where the value of an attribute can contain entity references, 
051     * the child nodes of the <code>Attr</code> node may be either 
052     * <code>Text</code> or <code>EntityReference</code> nodes (when these are 
053     * in use; see the description of <code>EntityReference</code> for 
054     * discussion). Because the DOM Core is not aware of attribute types, it 
055     * treats all attribute values as simple strings, even if the DTD or schema 
056     * declares them as having tokenized types. 
057     * <p>See also the <a href='http://www.w3.org/TR/2000/REC-DOM-Level-2-Core-20001113'>Document Object Model (DOM) Level 2 Core Specification</a>.
058     */
059    public interface Attr extends Node {
060        /**
061         * Returns the name of this attribute.
062         * 
063         * @return the name of the attribute.
064         */
065        public String getName();
066    
067        /**
068         * If this attribute was explicitly given a value in the original 
069         * document, this is <code>true</code>; otherwise, it is 
070         * <code>false</code>. Note that the implementation is in charge of this 
071         * attribute, not the user. If the user changes the value of the 
072         * attribute (even if it ends up having the same value as the default 
073         * value) then the <code>specified</code> flag is automatically flipped 
074         * to <code>true</code>. To re-specify the attribute as the default 
075         * value from the DTD, the user must delete the attribute. The 
076         * implementation will then make a new attribute available with 
077         * <code>specified</code> set to <code>false</code> and the default 
078         * value (if one exists).
079         * <br>In summary:  If the attribute has an assigned value in the document 
080         * then <code>specified</code> is <code>true</code>, and the value is 
081         * the assigned value.  If the attribute has no assigned value in the 
082         * document and has a default value in the DTD, then 
083         * <code>specified</code> is <code>false</code>, and the value is the 
084         * default value in the DTD. If the attribute has no assigned value in 
085         * the document and has a value of #IMPLIED in the DTD, then the 
086         * attribute does not appear in the structure model of the document. If 
087         * the <code>ownerElement</code> attribute is <code>null</code> (i.e. 
088         * because it was just created or was set to <code>null</code> by the 
089         * various removal and cloning operations) <code>specified</code> is 
090         * <code>true</code>.
091         * 
092         * @return <code>true</code> if the attribute was specified,
093         *         <code>false</false> otherwise.
094         */
095        public boolean getSpecified();
096    
097        /**
098         * Returns the value of the attribute is returned as a string. 
099         * Character and general entity references are replaced with their 
100         * values. See also the method <code>getAttribute</code> on the 
101         * <code>Element</code> interface.
102         * 
103         * @return the attribute value as a string.
104         */
105        public String getValue();
106        
107        /**
108         * Sets the value of the attribute. This creates a <code>Text</code> node
109         * with the unparsed contents of the string. I.e. any characters that an
110         * XML processor would recognize as markup are instead treated as literal
111         * text. See also the method <code>setAttribute</code> on the
112         * <code>Element</code> 
113         * interface.
114         * 
115         * @param value the new attribute value.
116         * 
117         * @exception DOMException 
118         *   NO_MODIFICATION_ALLOWED_ERR: Raised when the node is readonly.
119         */
120        public void setValue(String value)
121                                throws DOMException;
122    
123        /**
124         * The <code>Element</code> node this attribute is attached to or 
125         * <code>null</code> if this attribute is not in use.
126         * 
127         * @return the owner <code>Element</element>, or <code>null</code> is the
128         *         attribute is unused.
129         * 
130         * @since DOM Level 2
131         */
132        public Element getOwnerElement();
133    
134    }
